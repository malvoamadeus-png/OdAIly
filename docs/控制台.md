# 控制台

## 职责

控制台负责维护人工可配置的数据和复盘视图，不承载 worker 内部运行逻辑。

当前控制台是 Vite 前端，直接连接 Supabase。运行 worker 通过数据库表和通知机制读取配置变化。

## 访问控制

- 控制台使用 Supabase Auth 的邮箱密码登录。
- 前端仍使用 `VITE_SUPABASE_URL` 和 `VITE_SUPABASE_ANON_KEY` 初始化客户端，但控制台业务表不再对 `anon` 开放。
- 只有 `authenticated` 会话且邮箱存在于 `console_admins` 白名单中的用户，才能读取或修改控制台数据。
- 控制台不开放前端自助注册；管理员账号由 Supabase Auth 后台创建，再通过本地 CLI 加入白名单。

## 模块索引

- `控制台-X抓取.md`：X 抓取账号、频率、并发、抖动和启停。
- `控制台-非主流媒体抓取.md`：外媒站点、统一频率与抖动；前端按写作链路 / 标题提醒链路分组展示。
- `控制台-Prompt.md`：Prompt 模板、版本、删除、发布/回滚和特色模式开关。
- `控制台-竞品配置.md`：竞品排除词维护。
- `控制台-事件复盘.md`：事件列表、收藏、备注和详情。
- `控制台-发布者.md`：发布者总开关、北京时间窗、渠道启停和固定分类只读展示。

后续如果控制台新增较大的功能模块，应继续按子文档拆分，避免单个控制台文档过大。

## 数据边界

控制台可以写入配置类和复盘类表：

- `x_capture_settings`
- `x_capture_accounts`
- `non_mainstream_media_settings`
- `non_mainstream_media_sources`
- `prompt_templates`
- `prompt_template_versions`
- `publisher_settings`
- `publisher_channels`
- `competitor_filter_keywords`
- `newsflash_event_favorites`
- `newsflash_event_notes`
- `newsflash_item_notes`

控制台不直接推进任务状态，不替代 worker 的锁、重试和失败处理。

## 配置生效

配置变更的基本要求：

- worker 启动时主动加载当前配置。
- 控制台发布或保存配置后，由数据库通知 worker 刷新本地缓存。
- worker 不依赖高频轮询发现配置变化。

不同模块的通知机制见对应控制台子文档。

## 本地开发

```powershell
cd frontend
npm install
npm run dev
```

前端需要：

```text
VITE_SUPABASE_URL=
VITE_SUPABASE_ANON_KEY=
```

首次启用控制台登录时，还需要：

1. 在 Supabase Dashboard 的 Authentication 中创建管理员用户，邮箱与白名单保持一致。
2. 在 Supabase Dashboard 中把 Site URL / Redirect URL 指向控制台正式域名，例如 `https://od-a-ily.vercel.app`。
3. 在本地用数据库直连权限执行：

```powershell
python backend\src\main.py console-grant-admin --email your-admin@example.com
```

常用管理员白名单命令：

```powershell
python backend\src\main.py console-list-admins
python backend\src\main.py console-revoke-admin --email your-admin@example.com
```

## 巨鲸控制台

- 控制台新增一级标签 `巨鲸`，页面内包含 `链上` 与 `Hyperliquid`。
- `链上` 负责维护 EVM 地址和自定义标签，支持启用、停用、删除，并展示 worker 写回的链状态与最近播报。
- `Hyperliquid` 负责维护 Hyperliquid 地址和自定义标签，支持启用、停用、删除，并展示最近轮询状态与播报历史。
- `Hyperliquid` 还提供三个人工可修改参数：`单次门槛`、`聚合门槛`、`聚合时长`。
- 新增控制台数据表：`whale_watch_addresses`、`whale_watch_chain_states`、`whale_watch_activities`、`whale_watch_hyperliquid_settings`、`whale_watch_hyperliquid_addresses`、`whale_watch_hyperliquid_states`、`whale_watch_hyperliquid_activities`。
- 前端只写地址配置和 Hyperliquid 阈值配置；链选择、轮询、活动识别、滚动聚合窗口计算和 Telegram 发送分别由 `whale-watch-worker` 与 `whale-watch-hyperliquid-worker` 负责。
- 模块细节见 `docs/巨鲸.md`。
