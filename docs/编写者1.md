# 编写者1

## 职责

编写者1负责根据任务来源和判断者路由选择 Prompt，调用 AI 模型生成快讯初稿，并记录实际使用的 Prompt 模板和版本。

编写者1只生成内容初稿，不负责最终格式标准化和后台推送。

## 输入

编写者1处理已经通过搜索者和判断者的任务。

编写者1调用模型前仍执行统一时效兜底。`tasks.published_at` 缺失或超过 `PROCESSING_FRESHNESS_WINDOW_SECONDS` 窗口时，任务进入 `expired`，不调用写作模型。

输入主要来自：

- `tasks`：来源、标题、正文、来源链接、发布时间、原始 payload。
- `x_task_pipeline`：判断者 route、搜索者结果、候选事件关系。
- `prompt_templates` 和 `prompt_template_versions`：当前生效 Prompt。

## Prompt 选择

当前 X 和竞品快讯按判断者 route 选择模板：

- `regular` -> `x_regular_writer`
- `onchain` -> `x_onchain_writer`
- `funding` -> `x_funding_writer`
- `non_mainstream_media` -> `non_mainstream_media_writer`

`discard` 任务不进入编写队列。

非主流媒体来源的约束是：

- 先经过搜索者去重，再进入判断者。
- 判断者只负责判断是否废话，不再细分成 `regular / onchain / funding`。
- 只要判断者输出 `non_mainstream_media`，就统一使用 `non_mainstream_media_writer`。

`non_mainstream_media_writer` 的初始 seed 文件来自 `docs/外媒模板.txt`，并通过 Prompt 控制台维护版本。

Prompt 内容由控制台管理，初始模板从 `docs/*.txt` seed 到数据库。Prompt 管理只控制内容、版本和模板级 `特色模式` 开关，不控制模型、超时、重试或输出 schema。

当 `prompt_templates.feature_mode_enabled = true` 时，编写者1在调用写作模型前，会机械地在实际 Prompt 最前面补上一行 `开启特色模式`，再拼接原始任务内容。该开关不改写 `prompt_template_versions.content` 原文。

## 版本追溯

每次生成快讯时，需要记录：

- 使用的 Prompt template key。
- 使用的 Prompt version id。
- 使用的 worker 模块。
- 模型输出原文或必要的结构化结果。

这样后续可以回溯质量问题和 Prompt 版本影响。

## 运行配置

当前编写模型默认使用 `X_PROCESS_WRITER_MODEL`，线上默认值为 `gpt-5.5`。
编写者1调用 OpenAI 写作时会传入 `X_PROCESS_WRITER_REASONING_EFFORT`，当前线上默认值为 `high`。

相关命令：

```powershell
python backend\src\main.py x-process-worker --stage write
python backend\src\main.py x-process-worker --stage write --once
```

## 输出

编写者1输出快讯标题和正文初稿，并将任务推进到格式化阶段。

最终标题和正文仍需要经过编写者2的确定性格式化规则处理。
