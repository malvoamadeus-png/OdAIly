# 控制台-事件复盘

## 职责

控制台-事件复盘用于按新闻事件查看 Odaily 与竞品快讯的发布情况，支持筛选、收藏、备注和详情查看。

系统只提供事件聚合、来源归属、筛选、展示、收藏和备注能力。内容质量判断由用户人工完成，不做 AI 辅助点评。

## 数据来源

事件复盘使用：

- `newsflash_items`：来源快讯。
- `newsflash_events`：事件。
- `newsflash_event_sources`：事件与来源快讯关系。
- `newsflash_event_summary`：控制台列表视图。
- `newsflash_event_favorites`：收藏。
- `newsflash_event_notes`：事件备注。
- `newsflash_item_notes`：来源条目备注。

embedding 只保存在服务器本地 SQLite 缓存，不写入 Supabase。

## 列表视图

主表按事件展示，一行代表一个 `event_id`，不直接按单条快讯铺开。

主表字段包括：

- 事件时间。
- 事件代表标题。
- Odaily 发布情况。
- BlockBeats 发布情况。
- PANews 发布情况。
- 金色财经发布情况。
- 共同发布数量。
- 首发媒体和首发时间。
- 事件状态。
- 收藏状态。

## 快捷筛选

控制台提供一键筛选：

- 全部。
- 多家共同发布。
- 我们也发布。
- 高价值对比。
- 竞品有我方无。
- 竞品共识但我方无。
- 仅我方发布。
- 我方晚发。
- 我方首发。
- 已收藏。

## 详情与备注

事件详情懒加载。用户点击事件后，再请求该事件下的完整快讯明细。

详情区展示：

- `event_id`。
- 事件代表标题。
- 首发媒体和首发时间。
- 参与媒体数量和媒体列表。
- 按发布时间排序的来源快讯列表。
- 每条快讯的标题、正文、发布时间和来源链接。
- 事件备注和来源条目备注。

收藏和备注写入 Supabase，和本地 embedding 缓存分离。
