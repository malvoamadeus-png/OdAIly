# 收集者-金十

## 职责

收集者-金十负责：

- 定时轮询金十快讯接口
- 首次启用时只 seed 当前列表，不回灌历史快讯
- 按 `source_item_id` 去重
- 将新快讯写入 `tasks.source = 'jin10'`
- 为新快讯提交本地流水线 `write_flow` job
- 写入 `pipeline_worker_heartbeats.component = 'jin10_monitor'`

当前实现包为 `backend/packages/jin10_monitor/`，命令名为 `jin10-monitor-worker`。

## 接口配置

默认 endpoint：

```text
https://4a735ea38f8146198dc205d2e2d1bd28.z3c.jin10.com/flash
```

默认 headers 包含：

```text
x-app-id: bVBF4FyRTn5NJF5n
x-version: 1.0.0
referer: https://www.jin10.com/
origin: https://www.jin10.com
```

接口来源是金十网页端免费快讯接口线索，不依赖金十开放平台 key。由于该入口可能受网络环境、DNS 或风控影响，endpoint、headers、channel、interval 均写入 `jin10_settings` 并允许在控制台调整。

接口失败不会阻断其他收集链路；worker 只记录 `last_error` 和失败 heartbeat，下一轮继续重试。

## 存储边界

### Supabase / Postgres

- `jin10_settings`：全局开关、频率、endpoint、channel、headers、最近轮询和错误。
- `jin10_seen_items`：首次 seed 和后续去重用的 `source_item_id` 集合。
- `tasks`：新快讯正式入库，`source = 'jin10'`。
- `pipeline_worker_heartbeats`：收集器心跳。

`jin10_settings.singleton_key` 固定为 `global`。

### 本地流水线

新金十任务提交到本地 `local_pipeline`：

```text
judge_jin10 -> search -> write -> format_publish -> publish
```

本地 job 唯一键仍是 `source + source_item_id`。如果提交本地流水线失败，该 item 不会标记为已看，下一轮会再次尝试。

## Payload 解析

解析器支持常见字段：

- `id`
- `title`
- `content`
- `time`
- `url`

也支持金十快讯常见的嵌套结构，例如外层 `data` 中再包含 `data.content`。缺少 `id` 时会用标题、正文、时间等字段生成稳定哈希作为 `source_item_id`。

## 首次启用

`jin10_seen_items` 为空时，worker 会把当前接口返回的所有 item 标记为 `seeded = true`，但不会写入 `tasks`，也不会提交本地流水线。

之后发现的新 item 才进入正式写作发布链路。

## 判断与发布

金十任务先进入 `judge_jin10`：

- Prompt key：`jin10_judge`
- 输出只允许 `publish` 或 `discard`
- 通过后写入 `news_type = 'regular'`
- 丢弃写入 `discard_type = 'off_topic'`

金十发布渠道为 `publisher_channels.channel_key = 'jin10'`，默认关闭，所以通过判断并完成写作后默认进入后台审核，不自动发布。

## 运行命令

```powershell
python backend\src\main.py jin10-init-db
python backend\src\main.py jin10-monitor-worker --once
python backend\src\main.py jin10-monitor-worker
```

生产 systemd service：

```text
deploy/odaily-jin10-monitor.service
```
