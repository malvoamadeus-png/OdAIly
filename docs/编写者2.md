# 编写者2

## 职责

编写者2对应 `x_processing` 的 `format_publish` 阶段，负责：

- 规范标题和正文格式
- 产出最终定稿 `final_title` / `final_content`
- 把任务状态推进到 `publisher_pending`

编写者2不再负责：

- 调用发布接口
- 判断是否自动发布
- 发送统一编写链路的内部 Telegram 通知

## Crypto信源与 AI信源发布约定

- 所有进入编写链路的 Crypto信源和 AI信源稿件都保留 `sourceUrl`
- 来源名优先使用 `metadata.site_display_name`
- Crypto信源路线名显示为 `Crypto信源`
- AI信源路线名显示为 `AI信源`

这同时适用于：

- `tasks.source = 'non_mainstream_media'`
- `tasks.source = 'ai_source'`
- 历史兼容 `tasks.source = 'mainstream_media'`

## 仍然隐藏来源的类型

- `blockbeats`
- `panews`
- `jinse`
- `jin10`

竞品和金十来源继续不向后续发布接口透出原始链接。

## 与发布者的边界

- `format_publish` 成功后，后续是否自动发布由 `publish` 阶段单独判断。
- `publish` 阶段才会调用推送接口，并保证每条稿件只调用一次。
- 统一编写链路的内部 Telegram 通知也移动到 `publish` 阶段；无论最终是自动发布还是挂后台，通知都在发布者完成推送调用后发送。
