# 编写者2

## 职责

编写者2负责对编写者1生成的标题和正文做确定性标准化、格式化处理，并将快讯以 `isPublish=false` 状态推送至后台。

编写者2不调用 AI 模型。

## 输入

输入来自编写者1生成的快讯初稿：

- 标题。
- 正文。
- 来源。
- 原始来源链接。
- 判断者 route 和 Prompt 追溯信息。

编写者2在格式化和推送后台前仍执行统一时效兜底。`tasks.published_at` 缺失或超过 `PROCESSING_FRESHNESS_WINDOW_SECONDS` 窗口时，任务进入 `expired`，不调用 Push API，不发送 Telegram。

## 输出

输出为后台可审核的快讯：

- `title`：格式化后的标题。
- `content`：格式化后的正文。
- `isPublish=false`。
- `sourceUrl`：X 和主流外媒来源可传原始链接；竞品和非主流媒体来源推送后台时默认保持为空。

推送成功后，任务进入 `ready_review`。如果配置了 Telegram，则同步发送内部提醒。外媒标题提醒链路不走这里，单独由 `提醒者.md` 负责。

Telegram 配置：

- `TELEGRAM_BOT_TOKEN`：Telegram bot token。
- `TELEGRAM_CHAT_ID`：目标群组或频道 ID。
- `TELEGRAM_MESSAGE_THREAD_ID`：可选，Telegram forum topic 的 `message_thread_id`。配置后，编写者2提醒发送到指定 topic；留空时发送到群组默认话题。

编写者2的 Telegram 提醒统一使用以下格式：

- 第一行：`{来源名}有新快讯-{路线}-{模式状态}：{title}`
- 第二行：如果任务有原始链接，则附上原始 `source_url`

其中：

- `来源名` 继续沿用现有来源展示逻辑：X 使用 `X平台`，竞品使用业务名，非主流媒体使用 `非主流外媒`，主流外媒优先使用站点显示名。
- `路线` 使用中文业务名：`常规 / 链上 / 融资 / 非主流外媒 / 主流外媒`；如果当前 pipeline 缺少可用 route，则回退为 `未分类`。
- `模式状态` 只取编写者1写作当次实际使用的模式状态：开启时显示 `特色模式`，关闭或缺失时显示 `标准模式`。
- `模式状态` 不回查发送时的当前 Prompt 配置，而是读取写作阶段已落库到 `x_task_pipeline` 的实际值，避免写作后、发送前控制台切换开关导致通知文案与实际稿件不一致。

外媒标题提醒链路使用独立文案 `外媒标题提醒：{site_display_name}｜{title}`，不由编写者2发送。

## 固定格式

正文固定前缀为：

```text
Odaily星球日报讯
```

即“Odaily星球日报讯”后空一格再接正文。

| 标题及正文 | Binance | 币安 |  | Binance发布公告 | 币安发布公告 |

| 正文 | 段落末尾无标点 | 加上句号 |  | 该项目已完成融资 | 该项目已完成融资。 |

## 运行命令

```powershell
python backend\src\main.py x-process-worker --stage format_publish
python backend\src\main.py x-process-worker --stage format_publish --once
python backend\src\main.py telegram-test --message-thread-id 123 --text "OdAIly topic test"
```
