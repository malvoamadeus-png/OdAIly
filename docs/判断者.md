# 判断者

## 职责

判断者负责对候选内容做第一道轻量判断：先判断是否具备写成新闻快讯的基本价值，再把可处理内容路由到对应快讯类型。

判断者不是事实查证者，也不做深度去重。事实补充、去重和写作由后续搜索者和编写者承担。

## 输入

当前主要输入是 `tasks` 表中的 X 或竞品候选内容。

X 来源先进入判断者。竞品来源先进入搜索者，未重复时再进入判断者。

处理 worker 在调用判断模型前会先执行统一时效兜底。`tasks.published_at` 缺失或已超过 `PROCESSING_FRESHNESS_WINDOW_SECONDS` 窗口时，任务直接进入 `expired`，不会调用判断模型。

## 输出

判断者输出结构化 JSON：

```json
{
  "route": "regular | onchain | funding | discard",
  "discard_type": "none | pure_emotion | baseless_trading_call | daily_chatter | non_crypto_ai"
}
```

`route` 是核心输出：

- `regular`：进入常规快讯模板。
- `onchain`：进入链上快讯模板。
- `funding`：进入融资快讯模板。
- `discard`：不进入搜索者和编写者流程，任务标记为跳过。

`discard_type` 只在 `route=discard` 时有业务意义；进入后续流程时固定为 `none`。

判断者不输出 `confidence`、`template_key`、`reason`、`entities` 等解释性字段。`route` 与 Prompt 模板的映射由代码固定维护。

## 丢弃类型

- `pure_emotion`：纯情绪表达，没有可报道事实。例如“ETH 要起飞了”“太牛了”。
- `baseless_trading_call`：无逻辑的纯粹喊单、价格口号或无事实支撑的涨跌判断。
- `daily_chatter`：寒暄、日常状态、meme、节日祝福、无新闻事实的社区闲聊。
- `non_crypto_ai`：AI 泛科技内容，且不属于当前自动快讯流水线需要处理的 Crypto 新闻。

只要内容包含明确主体、明确动作和可报道结果，默认不要因为语气主观、营销语气或表达不规范而丢弃。

## 确定性预筛

判断者调用模型前先执行轻量确定性预筛。当前预筛只处理明显的 `non_crypto_ai`：

- 标题或正文命中 AI 主题词，例如 `AI`、`人工智能`、`大模型`、`生成式 AI`、`OpenAI`、`ChatGPT` 等。
- 同一内容没有命中 Crypto/Web3 语境词，例如 `Web3`、`Crypto`、`区块链`、`链上`、`代币`、`交易所`、`BTC`、`ETH`、`DeFi` 等。

命中时直接写入 `route=discard`、`discard_type=non_crypto_ai`，不调用判断模型，也不进入编写和推送流程。

如果 AI 新闻同时具有明确 Crypto/Web3 语境，例如 Web3 AI 项目融资、链上智能体、加密交易所 AI 产品等，不由预筛丢弃，继续交给判断模型路由。

## Prompt 要求

Prompt 必须明确：

- 任务分两步：先判断是否属于可丢弃内容，再判断 `regular`、`onchain`、`funding`。
- 四类丢弃内容的定义和例子。
- “无逻辑的纯粹喊单”和“有事实依据的市场事件”的区别。
- 只输出结构化 JSON，不输出解释文本。

Prompt 内容和版本由 Prompt 管理模块维护。模型、超时、重试和 API 风格由服务端运行配置维护。

## 运行配置

当前判断模型默认使用 `X_PROCESS_JUDGE_MODEL`，线上默认值为 `gpt-5.4-mini`。

相关命令：

```powershell
python backend\src\main.py x-process-worker --stage judge
python backend\src\main.py x-process-worker --stage judge --once
```

## 后续状态

- `route=discard`：任务不进入后续写作流程。
- 其他 route：任务进入搜索者或后续阶段，具体顺序由来源决定。

状态名和落库字段以当前 `x_task_pipeline` schema 和 `backend/packages/x_processing` 实现为准。
