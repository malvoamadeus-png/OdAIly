# 收集者-非主流媒体

## 职责

收集者-非主流媒体负责持续采集没有公开 API 的媒体站点内容，完成站点适配、基础清洗、站内去重和 `tasks` 入库。

当前它是统一入口，但会按站点固定分流到两条后续链路：

```text
write_flow:
收集者-非主流媒体 -> 搜索者 -> 判断者 -> 编写者1 -> 编写者2

alert_only:
收集者-非主流媒体 -> 判断者-领域分类 -> 搜索者 -> 提醒者
```

收集者本身不负责领域分类、不负责跨来源查重、不负责写作，也不负责 Telegram 提醒。

## 数据表与站点注册

当前配置与去重表：

- `non_mainstream_media_settings`：全局频率和抖动。
- `non_mainstream_media_sources`：后端已接入站点目录和启停状态。
- `non_mainstream_media_seen_items`：按站点记录已见过的列表发现项。

`non_mainstream_media_sources` 当前至少包含：

- `site_key`
- `display_name`
- `homepage_url`
- `capture_method`
- `pipeline_mode`
- `enabled`

同步规则：

- `python backend/src/main.py non-mainstream-media-init-db` 会初始化 schema 并同步站点目录。
- worker 启动时也会再次同步站点目录。
- 已有行保留 `enabled`，只覆盖只读元数据和 `pipeline_mode`。

## 抓取方式分层

后端站点注册表当前预留两种标准抓取方式：

- `html_request`：直接请求 HTML 或 RSS 后解析。
- `browser_render`：先用浏览器渲染，再提取内容。

第一版都走 `html_request`，但列表发现方式不同：

- `a16z_crypto_posts`：抓列表 HTML，筛 `/posts/article/*` 等内容页。
- `forbes_digital_assets`：抓 RSS feed，提取 `Digital Assets` 专栏详情页。
- `hk01_virtual_assets`：抓 issue 页 `__NEXT_DATA__`，提取详情页。
- `ft_crypto`：由于 `/crypto` 直抓会落到验证页，当前改走 Google News RSS 搜索 FT Crypto 标题，再解回 FT 原始链接。
- `wsj_*`：分别抓公开 RSS，提取标题和文章链接。
- `fortune_crypto`：抓 `section/crypto` 页 HTML，但该页会混入通用 `Latest Stories` 链接；当前只保留标题或 URL 明显命中 Crypto 关键词的文章链接。

## `write_flow` 入库契约

`write_flow` 站点仍按“列表发现详情页 -> 抓详情正文 -> 清洗全文 -> 入库”的方式工作。

固定写入：

- `tasks.source = 'non_mainstream_media'`
- `source_item_id = canonical_url`
- `source_url = canonical_url`
- `title = 文章标题`
- `content = 清洗后的详情页正文全文`
- `published_at = 来源发布时间`
- `status = 'pending'`

`metadata` 至少包含：

- `site_key`
- `site_display_name`
- `capture_method`
- `pipeline_mode = 'write_flow'`
- `content_format`
- `author_names`
- `tags`
- `categories`
- `excerpt`
- `canonical_url`
- `source_kind = 'non_mainstream_media'`

## `alert_only` 入库契约

`alert_only` 站点只做列表发现，不抓详情页正文。

固定写入：

- `tasks.source = 'external_media_alert'`
- `source_item_id = 列表发现到的文章 URL`
- `source_url = 列表发现到的文章 URL`
- `title = 列表标题`
- `content = excerpt / description；如果列表侧没有摘要，则回退到 title`
- `status = 'pending'`

`metadata` 至少包含：

- `site_key`
- `site_display_name`
- `capture_method`
- `pipeline_mode = 'alert_only'`
- `excerpt`
- `source_kind = 'external_media_alert'`

这条链路的 `tasks` 不依赖详情页抓取，因此当前不会补 `author_names`、全文正文或 `published_at`。

## 各站点规则

### a16z

- 列表页只负责发现内容 URL。
- 只保留 `/posts/article/*`、`/posts/podcast/*`、`/posts/videos/*`、`/posts/listicles/*`、`/posts/papers/*`。
- 过滤 `/posts/` 根页、`/posts/focus-areas/*`、`/posts/tags/*`。
- 详情页优先读 JSON-LD，正文会清理推荐阅读和法律免责声明尾段。

### Forbes

- 列表发现优先使用 RSS。
- 详情页标题、发布时间、作者优先读 `__NEXT_DATA__`。
- 正文优先读 `.article-body` / `.fs-article`。

### HK01

- 列表发现来自 issue 页 `__NEXT_DATA__ -> issue.blocks[*].articles[*]`。
- 详情页标题、描述、发布时间、作者、标签从 `article` 对象读取。
- 正文优先从 `article.blocks[*].htmlTokens` 提取。

### FT

- `/crypto` 主页在真实运行环境会返回验证页时，不再强依赖页面 HTML。
- 当前使用 Google News RSS 搜索 `site:ft.com crypto` 发现标题。
- 发现到的 Google News 项会进一步解回 `ft.com/content/*` 原始链接。
- 当前只做标题发现，不抓付费墙正文。

### WSJ

- 三个入口分开成三行站点记录。
- 当前通过公开 RSS 做标题发现，不抓正文详情。

### Fortune

- `section/crypto` 页本身会混入 Fortune 全站的 `Latest Stories`。
- 当前不会把该页上的所有文章链接都入库。
- 只保留标题或 URL 明显命中 Crypto 关键词的文章。
- 只做标题发现，不抓正文详情。

## 首轮 seed 和站内去重

首轮启动不回灌旧内容。

行为约定：

- 如果某站点 `seeded_at` 为空，第一轮只把当前发现项写进 `non_mainstream_media_seen_items`。
- 第一轮不创建 `tasks`。
- 完成第一轮后写入 `seeded_at`。
- 后续轮次只对新的列表发现项创建任务。

这里的去重只解决“同一站点同一列表发现项不要重复入库”。跨来源是否与 Odaily 或历史提醒重复，由后续搜索阶段处理。

## 运行命令

```powershell
python backend\src\main.py non-mainstream-media-init-db
python backend\src\main.py non-mainstream-media-worker
python backend\src\main.py non-mainstream-media-worker --once
```

注意：

- 这只初始化和运行收集者本身。
- `write_flow` 后续仍依赖 `x-process-init-db` / `x-process-worker`。
- `alert_only` 后续依赖 `external-media-alert-init-db` / `external-media-alert-worker`。
- 生产环境建议使用 `deploy/odaily-non-mainstream-media.service` 常驻运行收集者。

## 监控

收集者 worker 会把心跳写入 `pipeline_worker_heartbeats`，组件名固定为：

- `non_mainstream_media`

监督者会把它纳入心跳超时和近期无成功检查。
