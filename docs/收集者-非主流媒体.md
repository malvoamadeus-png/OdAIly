# 收集者-非主流媒体

## 职责

收集者-非主流媒体负责持续采集没有公开 API 的媒体站点内容，完成站点适配、基础清洗、去重和 `tasks` 入库。

这条线固定进入：

```text
收集者-非主流媒体 -> 搜索者 -> 判断者 -> 编写者1 -> 编写者2
```

它不负责事件复盘、不负责判断“是不是废话”、不负责改写和推送后台。

## 当前实现范围

当前已接入三个站点：

- `a16z_crypto_posts`
  - 列表入口：`https://a16zcrypto.com/posts/`
  - 抓取方式：`html_request`
- `forbes_digital_assets`
  - 列表入口：`https://www.forbes.com/sites/digital-assets/feed/`
  - 详情页入口：`https://www.forbes.com/sites/digital-assets/`
  - 抓取方式：`html_request`
- `hk01_virtual_assets`
  - 列表入口：`https://www.hk01.com/issue/10154/...`
  - 抓取方式：`html_request`

所有这类媒体统一写入：

- `tasks.source = 'non_mainstream_media'`

站点差异不通过 `tasks.source` 区分，而是放在 `metadata.site_key`、`metadata.site_display_name` 等字段中。

## 站点注册与配置

站点目录由后端代码维护，不由前端增删。

当前配置表：

- `non_mainstream_media_settings`：全局频率和抖动。
- `non_mainstream_media_sources`：后端已接入站点目录和启停状态。
- `non_mainstream_media_seen_items`：按站点记录已见过的详情页 ID。

同步规则：

- `python backend/src/main.py non-mainstream-media-init-db` 会初始化 schema 并同步站点目录。
- worker 启动时也会再次同步站点目录。
- 已有行保留 `enabled` 状态，只覆盖 `display_name`、`homepage_url`、`capture_method` 这类只读元数据。

## 抓取方式分层

后端站点注册表当前预留两种标准抓取方式：

- `html_request`：直接请求 HTML 后解析。
- `browser_render`：先用浏览器渲染，再提取内容。

第一版代码已经按这层切分：

1. 站点定义：`site_key / display_name / homepage_url / list_url / capture_method`
2. 页面获取：按 `capture_method` 选择抓取实现
3. 页面解析：列表发现详情页，详情页提取正文
4. 通用入库：去重、seed、写入 `tasks`

当前三个站点都走 `html_request`，但列表发现方式不同：

- `a16z_crypto_posts`：抓列表 HTML，筛 `/posts/article/*` 等内容页。
- `forbes_digital_assets`：抓 RSS feed，提取 `Digital Assets` 栏目的详情页 URL。
- `hk01_virtual_assets`：抓 issue 页 HTML，解析 `__NEXT_DATA__` 里的 `issue.blocks[*].articles[*]`。

## a16z 首版规则

列表页只负责发现详情页 URL。

当前保留的内容页路径：

- `/posts/article/*`
- `/posts/podcast/*`
- `/posts/videos/*`
- `/posts/listicles/*`
- `/posts/papers/*`

当前过滤掉：

- `/posts/` 根页
- `/posts/focus-areas/*`
- `/posts/tags/*`
- 其他不属于上述内容类型的 `/posts/*` 页面

对每个新 URL，会再抓详情页并解析：

- canonical URL
- 标题
- `datePublished`
- 作者
- 标签和栏目
- 正文全文

解析优先读取页面里的 JSON-LD，必要时回退到可见正文容器。

正文清洗会尽量移除：

- 导航、页头页脚
- 分享按钮和图片容器
- related/recommended/read more 一类推荐阅读块
- a16z 法律免责声明尾段

`tasks.content` 保存的是详情页清洗后的全文，不是列表摘要。

## Forbes 规则

Forbes 当前接的是 `Digital Assets` 专栏。

列表页不直接抓专栏 HTML，而是优先使用 RSS：

- feed：`https://www.forbes.com/sites/digital-assets/feed/`
- 只保留 `/sites/digital-assets/*` 详情页
- 自动去掉 query string，避免同一篇文章因追踪参数重复

详情页解析规则：

- 标题、发布时间、作者优先读 `__NEXT_DATA__`
- 正文优先读 `.article-body` / `.fs-article` 的可见段落
- 清洗时移除 newsletter signup 和语音播放器提示

## HK01 规则

HK01 当前接的是指定 issue 专题页，不是全站搜索。

列表页解析规则：

- 先抓 issue 页 HTML
- 从 `__NEXT_DATA__ -> props.initialProps.pageProps.issue.blocks[*].articles[*]` 发现详情页
- 详情页 URL 优先取 `publishUrl`，缺失时回退到 `canonicalUrl`

详情页解析规则：

- 标题、描述、发布时间、作者、标签从 `article` 对象读取
- 正文优先从 `article.blocks[*].htmlTokens` 提取
- 如果 `htmlTokens` 缺正文，再回退到可见正文容器

## 入库契约

收集者-非主流媒体写入 `tasks` 时的固定约定：

- `source = 'non_mainstream_media'`
- `source_item_id = canonical_url`
- `source_url = canonical_url`
- `title = 文章标题`
- `content = 清洗后的详情页正文全文`
- `published_at = detail page 的 datePublished`
- 不同站点发布时间字段不同，例如 Forbes 用 `date`，HK01 用 `publishTime`，统一写入 `published_at`
- `status = 'pending'`

`metadata` 至少包含：

- `site_key`
- `site_display_name`
- `capture_method`
- `content_format`
- `author_names`
- `tags`
- `categories`
- `excerpt`
- `canonical_url`
- `source_kind = 'non_mainstream_media'`

## 首轮 seed 和去重

首轮启动不回灌旧文章。

行为约定：

- 如果某站点 `seeded_at` 为空，第一轮只把当前发现的详情页写进 `non_mainstream_media_seen_items`。
- 第一轮不创建 `tasks`。
- 完成第一轮后写入 `seeded_at`。
- 后续轮次只对新的详情页 URL 创建 `pending` 任务。

这层去重只解决“同一站点同一详情页不要重复入库”，后续是否与 Odaily 或 in-flight 候选重复，由搜索者处理。

## 运行命令

```powershell
python backend\src\main.py non-mainstream-media-init-db
python backend\src\main.py non-mainstream-media-worker
python backend\src\main.py non-mainstream-media-worker --once
```

注意：

- 这只初始化和运行收集者本身。
- 主处理流水线仍需要已有的 `x-process-init-db` / `x-process-worker`。
- `x-process-init-db` 现在会同时 seed `non_mainstream_media_writer`。

## 监控

worker 会把心跳写入 `pipeline_worker_heartbeats`，组件名固定为：

- `non_mainstream_media`

监督者会把它纳入心跳超时和近期无成功检查。
