# 收集者-非主流媒体

## 说明

当前实现包名仍为 `non_mainstream_media`，但业务上它已经是统一外媒收集器。

它负责：

- 维护站点注册表
- 发现新文章或新标题
- 按 `pipeline_mode` 决定进入编写链路还是标题提醒链路
- 完成站内去重与 `tasks` 入库

worker 启动后会加载当前配置，之后不保留专门的数据库监听连接；运行中按固定窗口轮询重载配置，默认最多 5 分钟生效一次，不需要重启服务。

## 站点与模式

### `write_flow`

- `a16z_crypto_posts`
- `forbes_digital_assets`
- `hk01_virtual_assets`
- `coindesk`
- `cointelegraph`
- `decrypt`
- `tether_news`

这些站点会抓标题、来源链接、正文、作者等完整文章信息，并写入：

- `tasks.source = 'non_mainstream_media'`
- `tasks.status = 'pending'`

### Tether News 特例

- `tether_news` 的新文章发现优先使用公开 `WordPress REST API`：`https://tether.io/wp-json/wp/v2/posts?categories=3...`
- 单篇正文也优先通过 `wp-json/wp/v2/posts?slug=...&_embed=wp:term` 获取，不依赖 `/news/` 页面 DOM
- 如果生产 Linux 服务器访问 `tether.io` 原站遇到 Cloudflare `403 challenge`，收集器会自动 fallback 到 `https://r.jina.ai/http://...` 读取同一份官方列表 / 正文页面
- fallback 顺序是：官方 `wp-json` -> Jina 代理的 `wp-json` -> Jina 代理的 `/news/` 列表页；正文则是官方 `wp-json` 失败后切到 Jina 代理正文页
- 这样可以稳定拿到：
  - 标题
  - 正文 HTML
  - 发布时间
  - 分类 / 标签
  - 规范化文章链接
- 该站点接入 `write_flow`，和其他外媒全文一样直接生成 `tasks.source = 'non_mainstream_media'` 的待编写任务

### Cointelegraph 特例

- 新文章发现优先使用官方 `Google News sitemap`：`https://cointelegraph.com/sitemap/google-news.xml`
- 该源直接提供：
  - 标题
  - 详情链接
  - `news:publication_date`
- `https://cointelegraph.com/rss` 继续保留为 fallback，只在 sitemap 抓取失败或返回空列表时启用
- 发现层拿到的发布时间会透传到详情解析和 `tasks.published_at`
- 即使正文页缺少标准 `article:published_time`，也会优先沿用发现层时间，避免任务因缺少 `published_at` 被 freshness gate 直接过期

### `alert_only`

- `ft_crypto`
- `wsj_business`
- `wsj_economy`
- `wsj_finance`
- `fortune_crypto`
- `the_block`

这些站点只保留标题/摘要级材料，并写入：

- `tasks.source = 'external_media_alert'`
- `tasks.status = 'pending'`

## The Block 特例

- The Block 继续固定停留在 `alert_only`，只做标题提醒，不进入正文编写链路
- 新的标题发现优先级是：
  - 官方 `rss.xml`：`https://www.theblock.co/rss.xml`
  - `r.jina.ai` 代理后的官方 `rss.xml`
  - `Google News RSS` fallback：`https://news.google.com/rss/search?q=site:theblock.co&hl=en-US&gl=US&ceid=US:en`
- 这样做的原因是：Google News 对 The Block 存在明显收录延迟，可能导致标题提醒晚于原文发布时间数小时
- 服务器侧直连 `https://www.theblock.co/rss.xml` 仍可能遇到 Cloudflare / 403，但通过 `r.jina.ai` 代理读取官方 RSS 仍可稳定拿到：
  - 标题
  - 官方详情链接
  - `pubDate`
- The Block 正文抓取仍不可用，因此即使发现源切回官方 RSS，也只保留标题提醒用途

## 入库元数据

### `write_flow`

- `site_key`
- `site_display_name`
- `pipeline_mode = 'write_flow'`
- `author_names`
- `tags`
- `categories`
- `excerpt`
- `canonical_url`
- `source_kind = 'non_mainstream_media'`
- 如果正文页缺少发布时间但发现层有结果，会保留发现层 `published_at_raw`

### `alert_only`

- `site_key`
- `site_display_name`
- `pipeline_mode = 'alert_only'`
- `excerpt`
- `published_at_raw`
- `source_kind = 'external_media_alert'`
