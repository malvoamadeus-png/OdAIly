# 判断者-领域分类

## 职责

这是 `external_media_alert` 的 `domain_judge` 阶段。

它负责判断 Crypto信源标题提醒和 AI信源标题提醒是否属于 Crypto，并决定是否继续进入查重与提醒。

## 当前生产输入

- `tasks.source = 'external_media_alert'`
- `tasks.source = 'ai_source_alert'`

主要来自统一 Crypto信源收集器中的 `alert_only` 站点：

- FT Crypto
- WSJ Business / Economy / Finance
- Fortune Crypto
- The Block

AI信源标题提醒当前没有生产站点，但链路已保留 `ai_source_alert` 输入合同；提示词中的任务类型会显示为“AI信源标题提醒”。

## 历史兼容输入

- `tasks.source = 'mainstream_media'`

这类任务只为历史数据保留兼容，不再由新的收集入口生成。

## 输出

- 保留：`route = 'crypto'`
- 丢弃：`route = 'discard'`

如果模型输出了 `{"route":"discard"}` 但漏掉 `discard_reason`，worker 会兼容按 `non_crypto` 处理，避免任务卡在 `domain_failed`。

## Prompt

- `template_key = 'external_media_alert_domain_judge'`
- 需要同时兼容标题提醒和历史 `mainstream_media` 快讯任务
