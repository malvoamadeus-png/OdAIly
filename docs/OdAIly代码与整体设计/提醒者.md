# 提醒者

## 职责

提醒者只负责 Crypto信源和 AI信源标题提醒链路的最终入库与信息流展示状态。生产中由 `local_pipeline` 在本地 `alert_only` job 内调用，不再拆成独立 `external_media_alert@notify` 常驻 worker。

它不发送正常标题提醒 Telegram；值班侧通过信息流插件查看。Telegram 只由监督者发送错误、积压、心跳告警。

## 输入

- `tasks.source in ('external_media_alert', 'ai_source_alert')`
- `tasks.status = 'deduped'`
- `metadata.site_display_name`
- `title`
- `source_url`

`mainstream_media` 只保留历史兼容，不再作为新的提醒入口。

## 运行交接

- `non_mainstream_media` 对 `alert_only` 站点写入 `tasks` 后，提交本地流水线 `alert_only` job。
- `local_pipeline` 在同一进程内依次调用 `domain_judge -> search -> notify`。
- `external_media_alert_pipeline` 继续保存领域判断、查重和通知结果；正常标题提醒的 `telegram_result` 固定为 `skipped=true`，表示不发 Telegram；`tasks.status` 继续保留 `notified / duplicate / discarded / notify_failed` 等观察状态。

## 输出

- 不发送正常 Telegram 标题提醒
- 回写 `external_media_alert_pipeline.telegram_result`
- 更新 `tasks.status = 'notified'`

## 提醒文案

- 来源名优先使用站点显示名，例如 `The Block`
- Crypto信源标题提醒前缀显示为 `Crypto信源标题提醒`
- AI信源标题提醒前缀显示为 `AI信源标题提醒`
- 路线名固定由领域判断链路提供
- The Block 现在走这条链路，只提醒标题，不抓正文
