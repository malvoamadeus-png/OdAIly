# 控制台-X抓取

## 职责

控制台-X抓取是控制台一级标签“信源管理”里的 `X` 子页，用于配置收集者-X 的抓取规则，让 worker 启动时加载配置，并在运行中按固定窗口重载变更。

## 配置项

全局配置写入 `x_capture_settings`：

- 全局抓取频率。
- 最大并发数。
- 抖动时间。

账号配置写入 `x_capture_accounts`：

- X 用户名或用户主页 URL。
- 展示名。
- 写作名。
- 单账号抓取频率。
- 启用或停用状态。
- `AI信源` 标记。

所有账号默认使用全局频率；单账号配置了频率时，优先生效单账号频率。

`AI信源` 标记规则：

- 写入 `x_capture_accounts.is_ai_source`。
- 不改变收集者-X 的抓取行为，不改变 `tasks.source = 'x'`。
- 会让自动化主链路进入判断者-AI；保留后统一进入 X 常规写作模板。
- Chrome 插件 `信息流` 会把被标记账号的新快讯展示在 AI区，而不是高频区。

`写作名` 规则：

- 以账号 `username` 为匹配键维护。
- 仅用于 X 写作链路里的发言人名称标准化，不替代抓取侧真实作者元数据。
- 自动化 X 主链路与插件 `快讯生成` 共用这一个字段。
- 当 `写作名` 为空时，后续写作链路仍按原始作者显示名和用户名回退。

## 生效机制

收集者-X：

- 启动时加载 `x_capture_settings` 和 `x_capture_accounts`。
- 运行中不再保留专门的 Postgres 监听连接。
- 默认每 5 分钟内至少重载一次配置。
- 不需要重启服务。

## 控制台展示

`信源管理 > X` 子页应展示：

- 当前全局配置。
- 账号列表和启停状态。
- 每个账号的 `显示名` 与 `写作名`。
- 每个账号是否标记为 `AI信源`。
- 每个账号最近抓取时间、最近成功时间和最近错误。
- 最近 `x_capture_attempts`。
- 最近写入的 X `tasks`，按表格展示：
  - `收集到什么`：标题或正文首句
  - `来自哪里`：优先显示统一作者名 / 原始作者名 / 用户名
  - `如何处理了`：展示 `tasks.status` 的中文状态文案和入库时间

## 刷新策略

- X 抓取页打开时拉取一次最新数据。
- 顶部刷新按钮只刷新当前页。
- 不做自动轮询，避免控制台持续增加 Supabase 读取流量。
