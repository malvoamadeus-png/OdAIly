# 控制台-流程展示

## 职责

控制台“流程展示”是纯展示页，用于给管理员快速理解当前自动化主链路的信息流向。

它展示：

- 当前有哪些主工作流。
- 每条工作流从来源、收集、分流、查重、判断、写作、定稿到发布或提醒的节点顺序。
- 每个节点的职责概括、业务含义和产出。
- 搜索者的查重口径，包括“规则”“向量”“AI复核”分别在做什么。
- 已发布内容如何进入审核者的发布后异步质检。
- 明确使用 AI 模型的节点、模型名和思考等级。
- 极简流向视图，只看节点和箭头，不看详细说明。

它不负责：

- 写入 Supabase 配置。
- 调用后端接口。
- 推进任务状态。
- 替代 worker 的锁、重试、失败处理或发布规则判定。

## 展示范围

主画布只展示当前主发布与提醒链路：

- X 主线。
- Crypto 信源全文。
- Crypto 标题提醒。
- AI 信源全文。
- AI 标题提醒。
- 混合信源分流。
- 竞品快讯。
- 金十。

`巨鲸`、Chrome 插件不放入主画布；`审核者` 也不插入主发布链路，但会在页面尾部作为“发布后异步质检”明确说明。

## 视图

### 全景详细

默认视图。每条工作流展示：

- 入口 source 或配置口径。
- 默认状态流向。
- 分流说明。
- 节点职责。
- 节点背后的业务解释，例如 X 账号标签如何决定进入哪个判断者、搜索者的三段式查重分别是什么意思。
- AI 模型卡片。
- 节点产出。

### 极简流向

极简视图只展示节点顺序、箭头、关键分叉和 AI 模型徽标，便于快速扫读整条信息路径。

## 模型展示规则

- `judge_crypto`、`judge_ai`、`judge_jin10` 展示默认 `odaily-deepseek-review`，思考等级 `low`。
- `write` 展示默认 `odaily-gpt-writer`，思考等级 `low`。
- 混合信源收集器内分流展示 `odaily-deepseek-fast` 和“轻量分流”，不额外声明未固定的 reasoning effort。
- `search` 展示为“规则 + 向量 + 搜索者 AI 复核”。
- 其中“规则”指来源链接 / 标题等精确重复判断；“向量”指 embedding 相似度比较；“AI 复核”指搜索者自己的复核配置 `SEARCH_AI_REVIEW_*`，不复用判断者配置。
- `domain_judge` 展示为“按当前 alert 判断配置”，不编造默认模型。
- `format_publish`、`publish`、`notify` 默认按非固定模型节点展示。
