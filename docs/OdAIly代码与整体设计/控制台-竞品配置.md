# 控制台-排除词规则组

## 职责

控制台原“竞品配置”页面现在维护全来源的路径化排除规则组。每个规则组一次配置名称、说明、整组开关、适用路径、适用范围和多行词表，不对每个词单独选择路径或范围。前端编辑器里的 `termsText` 只是草稿字段，保存时会拆成数据库列 `terms text[]`，不会直接写入表结构。

## 数据边界

权威表为 `source_exclusion_rule_groups`：

- `rule_key`：稳定代码标识。
- `name`、`description`：控制台名称与说明。
- `scopes text[]`：整组适用路径。
- `terms text[]`：整组词表。
- `match_target text`：整组适用范围，`title` 表示仅标题类文本，`all` 表示标题、摘要与正文。
- `enabled`、创建与更新时间。

固定路径为 `x / competitor / crypto_source / ai_source / mixed_source / jin10`。表只允许 `authenticated + console_admins` 读写，匿名访问禁止。

旧 `competitor_filter_keywords` 不再被运行时代码读取，也暂不删除。显式初始化会把旧表启用和停用词分别迁入兼容规则组，默认 `match_target = title`；默认同时创建只作用于 `crypto_source` 的 `Ripple / XRP 商务稿`，词表为 `Ripple / XRP / RLUSD`。

## 控制台能力

页面采用三栏布局：规则组列表、整组编辑器、测试文本。支持新增、修改、启停、删除、按路径筛选和按名称 / 说明 / 词表搜索；词表通过批量 textarea 编辑，适用范围通过规则组级单选项维护。

测试文本仅在浏览器内做 NFKC、大小写不敏感的子串预览，不保存输入内容。测试区区分标题与摘要 / 正文，`title` 规则只检查测试标题，`all` 规则检查两者；它用于确认会命中哪些启用规则组，不代替后端最终匹配。

## 运行语义

后端统一执行 Unicode NFKC、大小写不敏感、任一词子串匹配，不检查 URL。`match_target = title` 只检查标题类文本；`match_target = all` 检查标题、摘要和正文。外媒先检查发现页标题 / 摘要，未命中再抓全文并复查；混合信源先检查 `mixed_source`，分类后再检查目标路径。

命中发生在创建 `tasks` 前：来源项目标记为已处理，停止后续查重、判断和写作。系统不创建任务、不写审计表、不保留命中标题、历史或计数，也不输出竞品逐条排除日志和过滤统计。
