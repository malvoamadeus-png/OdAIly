# 提醒者

## 职责

提醒者只服务外媒标题提醒链路，负责把“已经通过领域分类和查重”的标题提醒发送到 Telegram。

它不调用写作模板，不调用 Push Data API，也不写入后台待审核内容。

主流外媒快讯的完整生成链路不走这个 worker；那条链路在编写者2推送后台成功后发送内部提醒。

## 输入

提醒者读取：

- `tasks.source = 'external_media_alert'`
- `tasks.status = 'deduped'`

这些任务已经完成：

1. 收集
2. 是否 Crypto 的领域分类
3. 与 Odaily 和历史提醒的查重

## 输出

提醒者成功时：

- 发送 Telegram 消息
- 回写 `external_media_alert_pipeline.telegram_result`
- 把任务状态推进到 `notified`

失败时：

- 把任务状态推进到 `notify_failed`
- 记录失败原因

## Telegram 文案

提醒者与写作链路的 Telegram 提醒明确分开。

当前固定文案：

```text
外媒标题提醒：{site_display_name}｜{title}
{source_url}
```

第二行固定附原链接。

## 运行配置

当前默认使用：

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`
- `EXTERNAL_MEDIA_ALERT_TELEGRAM_MESSAGE_THREAD_ID`

如果没有单独配置 topic，则回退到 `TELEGRAM_MESSAGE_THREAD_ID`。

## 运行命令

```powershell
python backend\src\main.py external-media-alert-worker --stage notify
python backend\src\main.py external-media-alert-worker --stage notify --once
```

生产环境建议使用 `deploy/odaily-external-media-alert@.service`，实例名：

- `odaily-external-media-alert@notify.service`

## 状态推进

```text
deduped -> notifying -> notified
```

失败终态：

```text
notify_failed
```
