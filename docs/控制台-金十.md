# 控制台-金十

## 职责

控制台“金十”页签负责维护金十收集链路配置，并展示最近运行状态和最近入库任务。

控制台只写配置，不直接轮询接口、不推进任务状态、不提交本地流水线 job。

## 页面字段

配置写入 `jin10_settings.singleton_key = 'global'`：

- `enabled`：总开关。关闭后 `jin10-monitor-worker` 不请求接口、不入库。
- `interval_seconds`：常驻 worker 两轮之间的等待秒数。
- `endpoint_url`：金十快讯 HTTP endpoint。
- `channel`：可选 channel 参数；为空时不传。
- `request_headers`：请求 headers JSON。

状态展示来自同一行：

- `last_polled_at`
- `last_success_at`
- `last_error`
- `updated_at`

最近入库列表读取 `tasks.source = 'jin10'` 的最新任务，只用于复盘，不允许在此页面改任务状态。

## 默认值

默认关闭：

```text
enabled = false
interval_seconds = 60
```

默认 endpoint：

```text
https://4a735ea38f8146198dc205d2e2d1bd28.z3c.jin10.com/flash
```

默认发布渠道 `publisher_channels.channel_key = 'jin10'` 也为关闭状态。打开金十收集开关不等于打开自动发布；发布阶段仍会因为 `jin10` 渠道默认关闭而挂后台。

## 配置生效

保存配置后，数据库会发送 `jin10_config_changed` 通知。

常驻 worker 每轮都会读取最新配置；关闭开关后下一轮会进入 disabled 状态，只写 heartbeat，不请求金十接口。

## Prompt 管理

金十判断 Prompt 不在“金十”页签编辑，而是在“Prompt”页签维护：

- template key：`jin10_judge`
- 显示名：`判断者-金十`
- 种子文件：`docs/判断者-金十模板.txt`

该 Prompt 推荐用自然语言维护主题白名单，例如“美伊冲突、霍尔木兹海峡通行、和平协议拟定与美伊谈判”。
