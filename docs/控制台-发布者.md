# 控制台-发布者

## 页面职责

发布者页面只负责维护发布者运行配置，不承载发布决策逻辑本身。

前端当前提供：

- 发布者总开关
- 北京时间窗起止
- 渠道启停
- 固定分类只读展示

前端不提供：

- 发布者 Prompt 编辑
- 分类增删改
- 手工触发历史稿件补发

## 数据来源

- `publisher_settings`
- `publisher_channels`

页面仍走控制台统一的 Supabase Auth + `console_admins` 白名单权限。

## 发布者总开关

- 字段：`publisher_settings.enabled`
- 关闭后，所有进入发布者阶段的稿件仍然会调用一次推送接口
- 调用参数固定为 `isPublish=false`、`isPush=false`
- 总开关只控制“是否允许自动发布判断”，不跳过 `publish` 阶段

## 时间窗

- 时区固定只读展示为 `Asia/Shanghai`
- 可修改字段：
  - `window_start_local`
  - `window_end_local`
- 第一版默认值：`00:01` 到 `07:30`
- 时间窗只影响“当前是否允许自动发布”；不负责未来补发

## 渠道配置

当前渠道：

- `external_media`：外媒
- `x`：X
- `competitor`：竞品
- `jin10`：金十

第一版默认建议：

- `external_media = true`
- `x = false`
- `competitor = false`
- `jin10 = false`

实验样本按主任务来源计算；当前只建议把 `non_mainstream_media` 映射的外媒渠道纳入自动发布实验。

## 固定分类

页面只读展示以下分类 key：

- `policy_regulation`
- `people_view`
- `major_project_progress`
- `funding`
- `other`

说明：

- 前四类允许自动发布
- `other` 一律挂后台
- 如果未来要修改分类，必须改后端代码、前端只读文案和文档，不是控制台运行时配置

## 保存与生效

- 页面保存后写入 Supabase 表
- worker 读取配置后在后续新任务中生效
- 已经完成发布者决策的历史任务不会因为改配置而被自动重放
