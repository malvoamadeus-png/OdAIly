# 控制台-发布者

## 页面职责

发布者页面负责维护“是否允许自动发布”的规则配置。页面不直接做发布决策，保存后由服务器把前端规则转换成发布者 Prompt，后续新进入 `publish` 阶段的任务读取服务器本地配置生效。

页面分为两块：

- `常规`：承接未标记为 AI信源的 X、Crypto信源、竞品、金十等自动化写作链路。
- `AI信源`：承接 `tasks.source = 'ai_source'` 以及 `tasks.source = 'x'` 且 `tasks.metadata.x_account_is_ai_source = true` 的任务；页面展示整块，但置灰并标注 `暂未启用`，当前进入发布者时只挂后台，不自动放行。

每块内部都有：

- 发布者块级启用 / 停用开关
- `通过规则`
- `排除规则`

## 规则字段

规则大行字段：

- `规则名`
- `规则说明`
- `案例列表`
- `启用 / 停用`

新增规则默认停用。停用规则允许作为草稿保存；启用规则必须填写规则名和规则说明。案例只作为 Prompt 参考，不作为结构化输出字段。

规则不提供排序功能。当前规则数量较少，Prompt 拼接按保存时列表顺序输出。

## 判定契约

发布者模型输出是否通过和一小段业务理由：

```json
{
  "decision": "pass|reject",
  "reason": "一小段中文业务理由"
}
```

判定原则：

- 排除规则优先；只要命中任意启用排除规则，必须 `reject`。
- 未命中排除规则、且命中任意启用通过规则，才允许 `pass`。
- 未命中明确通过规则时默认 `reject`，进入后台忽视。
- `reason` 必须写给值班编辑看，说明这条快讯命中了哪些正文事实、对应哪类通过或排除规则、为什么直发或挂后台。
- 不再要求模型输出分类名，也不在 `reason` 里写内部原因码。

## 默认常规规则

默认 `常规` 发布者开启，并内置以下启用规则。

通过规则：

- `链上大额 / 巨鲸 / 清算`
- `政策 / 监管 / 法案`，要求与 Crypto、稳定币、交易所、DeFi、数字资产等直接相关
- `产品发布 / 升级 / 上线`，仅 Crypto 相关
- `预测市场 / 事件合约`，近期作为独立放行主题
- `宏观数据 / 利率 / 央行`，只放行关键宏观节点和明确利率政策信号
- `机构观点 / 人物表态`，仅放行“高重要性主体 + 直接 Crypto/监管/市场结构相关 + 含具体新信息/新动作/新数据”的观点
- `Crypto 公司重大股价 / 资本市场变动`，要求 Crypto 相关主体、显著股价或资本市场变化，以及明确公司动作或业绩/资产/产能/融资事实
- `融资`
- `安全 / 事故 / 停运`
- `项目 / 组织重大动向`

排除规则：

- `自我宣传 / 站台型表达`
- `地缘政治`
- `市场行情 / 价格异动`
- `非 Crypto 的宏大政治评论`
- `泛宏观 / 财政金融总量`
- `链上指标归纳 / 市场状态解读`
- `多主题市场综评 / 拼盘解读`
- `TRON / 孙宇晨相关`
- `中国领土 / 主权 / 安全红线`
- `营销活动 / 社区活动`
- `代币分发 / 空投 / 激励`
- `交易所网络升级充值提现暂停`
- `数字人民币`
- `非 Crypto 传统制造业供应链价格调整`
- `战略合作`
- `中小主体例行公司新闻`
- `小众传统金融产品异动`
- `冷门论文 / 估值模型 / 研究发布`

补充边界：

- 泛宏观、财政赤字、政府预算、货币市场基金规模、一般流动性总量等默认不发。
- “关键组织的关键人物”只有出现正式政策动作、监管动作、产品/市场结构变化或明确政策信号时才放行；工作组、研究框架、专家名单等默认不发。
- 高管离职不按公司名自动放行；非核心公众人物离任默认不发，除非正文明确证明影响核心产品、上市进程、监管资格、托管/清算安全或广泛用户资产。
- TRON、TRX、波场、火币、HTX、孙宇晨、Justin Sun、Sun Yuchen 相关内容默认挂后台。
- 非具体主体资金动作的链上指标归纳、市场状态解读默认挂后台。
- 多主题市场综评、行情拼盘或东一嘴西一嘴的解读稿默认挂后台；即使同时提到币价、技术指标、宏观数据、法案进展、美股或个股波动，只要没有围绕一个明确核心新闻事实展开，就不能按其中任一元素直发。
- 预测市场内容可按独立主题放行，但命中中国领土、主权、安全红线时排除规则优先。
- 交易所因支持网络升级、钱包维护、节点维护而暂停或恢复充值提现的公告默认挂后台，例如币安暂停 Moonbeam 网络充值提现。
- 数字人民币、e-CNY、数币钱包、数币试点、数币支付场景默认挂后台。
- 非 Crypto 的传统制造业零部件、材料、供应链、渠道价格调整默认挂后台；这不是 AI信源开关，AI/半导体核心产业链若未来要直发，应由 AI信源发布者或独立开关承接。

没有命中上述明确规则的内容不需要单独写 `其他` 规则；模型默认 `reject`。

## 保存与生效

保存路径：

```text
前端 -> 服务器保存接口 -> 服务器本地 JSON 配置生效 -> Supabase 当前快照写一次
```

具体接口：

- `POST /console/publisher-rules/get`
- `POST /console/publisher-rules/save`

接口由 `editor-plugin-api-server` 提供，但权限使用控制台 `console_admins` 白名单，不使用插件白名单。

运行时读取：

- worker 读取服务器本地 `data/config/publisher_rules.json`
- worker 不轮询 Supabase
- Supabase 表 `publisher_rule_config` 只保留当前版本快照，不保存版本历史
- 前端优先读取服务器接口；如果接口失败，会回退显示 Supabase `publisher_rule_config` 当前快照；快照也失败时再显示本地默认规则，并在页面提示加载告警
- 当默认规则版本升级时，服务器加载旧版本地 JSON 或 Supabase 快照会自动合并新增默认规则，并更新已有默认规则的说明和案例；自定义规则保留，已有规则启用 / 停用状态保留

## 旧配置

旧表仍保留：

- `publisher_settings`
- `publisher_channels`

当前发布者运行时不再读取旧总开关、旧渠道开关或旧北京时间窗；唯一生效开关是规则页里的 `常规` / `AI信源` 两个块级开关。
