# 完整程序架构

## 当前有效主链路

- `X`：`x_capture` 入库后进入 `x_processing(judge -> search -> write -> format_publish)`
- `竞品快讯`：直接进入 `x_processing(search -> write -> format_publish)`
- `外媒全文`：统一由 `non_mainstream_media` 收集，写入 `tasks.source = 'non_mainstream_media'`，进入 `x_processing(judge -> search -> write -> format_publish)`
- `外媒标题提醒`：统一由 `non_mainstream_media` 的 `alert_only` 站点写入 `tasks.source = 'external_media_alert'`，进入 `external_media_alert(domain_judge -> search -> notify)`

## 外媒系统级契约

- 业务上只保留两条外媒模式：`write_flow` 与 `alert_only`
- 内部仍复用旧 source：
  - `write_flow` -> `tasks.source = 'non_mainstream_media'`
  - `alert_only` -> `tasks.source = 'external_media_alert'`
- `mainstream_media` 只保留历史兼容，不再作为新的生产入口
- `external_media_alert_fetcher` 已退役，不再是生产必需 worker

## 统一外媒收集边界

- 实现包仍叫 `non_mainstream_media`，但业务上代表统一外媒收集器
- `write_flow` 当前站点：
  - a16z crypto Posts
  - Forbes Digital Assets
  - HK01 NFT / Virtual Assets
  - CoinDesk
  - Cointelegraph
  - Decrypt
- `alert_only` 当前站点：
  - FT Crypto
  - WSJ Business / Economy / Finance
  - Fortune Crypto
  - The Block

## 运行面

- 常驻收集：
  - `non_mainstream_media`
  - `x_capture`
  - `competitor_monitor`
- 外媒标题提醒处理：
  - `external_media_alert_domain_judge`
  - `external_media_alert_search`
  - `external_media_alert_notify`
- 通用内容处理：
  - `x_process_judge`
  - `x_process_search`
  - `x_process_write`
  - `x_process_format_publish`

## 控制台访问契约

- 控制台仍是浏览器直连 Supabase 的 Vite 前端，不新增独立控制台后端。
- 控制台访问依赖 Supabase Auth 邮箱密码登录；公开 `anon key` 只用于初始化客户端，不授予控制台数据权限。
- 控制台业务表统一通过 `authenticated + console_admins 白名单` 的 RLS 规则放行。
- 后端 worker 继续通过 `SUPABASE_DB_URL / DATABASE_URL` 直连 Postgres，不依赖控制台登录态，也不受前端会话过期影响。

## 定时行情简报契约

- `gate-tradfi` 与 `us-market` 共用 `odaily-worker` 调度进程，但日历规则分开维护。
- `gate-tradfi` 只按北京时间工作日执行：
  - `morning`：北京时间上午。
  - `open`：按美东开盘时点触发，但仍要求该触发时刻对应的北京时间是周一到周五晚间。
- `us-market` 按美东交易日执行：
  - `premarket`：只保留手动执行能力，不进入自动调度。
  - `open`：美东开盘。
  - `close`：北京时间次日早晨播报上一美东交易日收盘，因此整条链路从北京时间周一晚间开始，到北京时间周六早晨结束。
- `us-market` 的自动调度必须同时跳过美东周末和 NYSE 休市节假日；`gate-tradfi` 不复用这套美东周末判定。
