# 搜索者

## 职责

搜索者负责判断新任务是否与以下对象重复：

- Odaily 已发布参考语料
- 当前仍在流转中的 active 候选事件
- 外媒链路中，最近已通知过的标题提醒或已生成待审核的主流外媒快讯

搜索者只判断“是否同一事件/是否应去重”，不负责真假判断和改写。

## 适用链路

### `x_processing` 搜索阶段

用于：

- X 来源
- 竞品来源
- 非主流媒体全文写作来源

比对对象：

- `odaily_reference_items`
- active `search_event_candidates`

### `external_media_alert` 搜索阶段

用于外媒相关的轻量去重链路，包括：

- 外媒标题提醒任务 `tasks.source = 'external_media_alert'`
- 主流外媒快讯任务 `tasks.source = 'mainstream_media'`

比对对象：

- `odaily_reference_items`
- 已成功通知的标题提醒历史
- 已进入 `ready_review` 或 `notified` 的主流外媒快讯历史

## 存储边界

### Supabase / Postgres

作为持久化与共享副本，仍保存：

- `odaily_reference_items`
- `search_event_candidates`
- `search_event_sources`
- 外媒提醒任务及其历史状态

### 本地 SQLite

本轮搜索工作集统一放在：

- `data/processed/searcher/searcher.sqlite`

本地库同时承担两种角色：

- 文档镜像缓存
- embedding 缓存

当前会保存的高频文档类型包括：

- `odaily_reference`
- `candidate`
- `external_media_alert_history`
- 查询过程中的临时 `task`

其中 `external_media_alert_history` 当前既承载标题提醒历史，也承载主流外媒快讯历史窗口。

## local-first 读取规则

### Odaily 参考语料

- 先读本地 `searcher.sqlite`
- 本地为空时，再从 Supabase 读取最近窗口
- 回源结果会立刻写回本地，后续 steady-state 不再反复回源

### 活跃候选事件

- `x_processing` 搜索时优先读本地 active candidate
- 本地为空时才回源 Supabase，并回填本地
- active candidate 默认按“创建后 `24 小时`”计算有效期，超过窗口后不再作为 in-flight 去重依据
- candidate 必须带明确 `expires_at` 才会被视为 active；缺失过期时间的旧本地缓存不会继续参与去重

### 已通知提醒历史

- `external_media_alert` 搜索时优先读本地历史
- 本地为空时才回源 Supabase，并回填本地

### 主流外媒快讯历史

- `external_media_alert` 搜索主流外媒任务时，也使用同一份本地历史窗口
- 本地 recent window 不足时，回源集合会同时包含 `external_media_alert:notified` 和 `mainstream_media:ready_review/notified`

### 损坏恢复

- `searcher.sqlite` 打开失败或损坏时，会自动删除并重建
- 重建后通过最近窗口回源重新预热

## 本地镜像更新来源

本地文档镜像不是独立采集器维护，而是由现有写路径顺手刷新：

- 收集者-竞品在 Odaily 参考语料保存成功后，镜像到本地搜索库
- `x_processing` 在创建 primary candidate 后，把 active candidate 镜像到本地
- `x_processing` 在丢弃、失败、过期等终态时，把本地 candidate 标为 inactive
- `external_media_alert` 在通知成功后，把历史提醒镜像到本地

主流外媒快讯当前不在 `x_processing` 写回本地搜索历史；这部分历史由 `external_media_alert` 搜索阶段在 recent window 缺失时按需从 Supabase 回源并补进本地缓存。

因此 steady-state 下，搜索阶段主要读取本地，而不是把热语料从 Supabase 拉回 worker。

## 去重策略

当前仍是两段式判定：

1. 向量相似度快速筛选
2. 中间相似度区间时，交给 AI 做结构化复判

默认阈值：

- 直接重复阈值：`0.88`
- AI 复判起点：`0.78`

重复目标类型保持不变：

- `odaily_published`
- `inflight_candidate`
- `none`

其中外媒链路的历史去重结果会使用：

- `external_media_alert_history`

## in-flight 候选释放

active candidate 只应代表“仍可能继续流转的主任务”。

- 候选在创建时就会写入 `expires_at`，默认 TTL 为“创建后 `24 小时`”
- 这个 TTL 只作用于 `search_event_candidates` / 本地 `candidate` 缓存，不影响 Odaily 已发布参考语料或外媒历史窗口

当主任务进入这些状态时，本地 candidate 必须同步失活：

- `discarded`
- `expired`
- 任一失败终态

这样可避免旧 candidate 长时间占位，误伤后来的同事件任务。

## 心跳

搜索相关 worker 仍写 `pipeline_worker_heartbeats`，但采用统一降频策略：

- 稳定态最多 60 秒一条
- 状态变化立即写
- 失败立即写
