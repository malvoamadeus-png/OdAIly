# 监督者

## 职责

监督者是只告警的健康检查 worker。

它负责发现：

- worker 心跳超时
- worker 长时间没有成功工作
- claimable 任务积压
- processing 任务卡住
- 某类失败在最近窗口异常增多

它不会：

- 自动重启服务
- 修改任务状态
- 清理队列

## 依赖数据

监督者主要读取：

- `pipeline_worker_heartbeats`
- `pipeline_alerts`
- `tasks`
- `x_task_pipeline`
- `x_capture_attempts`

## 心跳语义

`pipeline_worker_heartbeats` 里有两个关键时间：

- `last_seen_at`：worker 最近一次上报
- `last_success_at`：worker 最近一次成功上报

本轮之后，所有主要 worker 共享同一降频策略：

- 稳定态最多 60 秒写一次心跳
- 状态变化立即写
- 失败立即写

因此监督者的判断口径是：

- “进程还活着”看 `last_seen_at`
- “最近确实做成过事”看 `last_success_at`

告警阈值没有缩短，默认仍高于 60 秒心跳间隔。

## X 收集健康判定

`x_capture_attempts` 现在会对空轮询 success 做 10 分钟采样，所以监督者不能再把“最近没有 success attempt”直接等同于故障。

当前 X 收集链路的健康判定是：

- 最近窗口内存在成功 `x_capture_attempts`，或
- 最近窗口内存在成功 `x_capture` heartbeat

只有两者都没有时，才触发 X 收集异常告警。

## 告警类型

当前主要告警包括：

- `stale_heartbeat:*`
- `stale_success_heartbeat:*`
- `old_claimable:*`
- `stuck_processing:*`
- `recent_failed:*`
- `x_capture:no_recent_success_attempt`

同一 `alert_key` 在去重窗口内不会重复刷屏。

## 运行命令

```powershell
python backend\src\main.py pipeline-supervisor --once
python backend\src\main.py pipeline-supervisor
```
