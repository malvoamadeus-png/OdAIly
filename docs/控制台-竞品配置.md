# 控制台-竞品配置

## 职责

控制台-竞品配置用于维护竞品和 Odaily reference 收集前的排除词。

排除词用于过滤不进入正式任务和事件复盘的快讯，例如价格突破、跌破、爆仓和指定品牌词。

## 数据边界

排除词写入 Supabase 表 `competitor_filter_keywords`。

字段以当前 schema 为准，核心含义包括：

- `term`：原始词。
- `term_normalized`：归一化词。
- `enabled`：是否启用。

## 控制台能力

控制台支持：

- 新增排除词。
- 修改排除词。
- 启用或停用排除词。
- 删除排除词。

## 生效范围

排除词匹配：

- `odaily`
- `blockbeats`
- `panews`
- `jinse`

排除词不匹配 X 来源。

匹配范围是清洗后的标题 + 正文。中文按原文子串匹配，英文大小写不敏感。

## 命中行为

命中排除词后：

- 不写入 `tasks`。
- 不写入 `odaily_reference_items`。
- 不进入 `newsflash_*` 事件复盘流程。
- 不进入搜索者、判断者、编写者和推送流程。
- 只写服务日志供排查。
