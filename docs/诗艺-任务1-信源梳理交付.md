## 一、本产出在系统中的位置

本次梳理**不是**编写者/发布者提示词工作，也**不是**重复统计已在追踪流里的账号。

| 系统模块 | 本次是否涉及 | 说明 |
|---|---|---|
| **收集者**（X 抓取 / 外媒站点） | ✅ 本次产出服务对象 | 产出是「建议新增进抓取名单」的候选信源 |

**工作流程**：

```

    → 南枳确认后，由开发加入 x_capture_accounts / non_mainstream_media_sources
```

## 二、核心发现（回应上次反馈）

### 发现一：编辑手搬最多的账号，全不在自动抓取名单里

对 `newsflash_items` 过去 60 天、带 `source_url` 的快讯做拆解，剔除竞品站内页和已接入站点后，排名前列的 X 账号中：

- **arkham** 被引用 30 次 → 我们用过 26 次、竞品 4 次
- **deitaone** 被引用 19 次 → 几乎全是我们的编辑手搬
- **peckshieldalert** 被引用 17 次
- **binancewallet** 被引用 15 次 → 全是 Alpha 上新/空投门槛

这些账号编辑天天在用，但 `x_capture_accounts` 里一个都没有。**本次梳理的核心价值：把「编辑已验证有效」的信源，转为机器自动跟踪。**

### 发现二：上次提交的问题出在哪

上次提交的 Lookonchain（488 次）、The Block（461 次）、CoinDesk（398 次）、ai_9684xtpa（86 次）等，**绝大多数已在追踪流程或属于人人皆知的大媒体**，统计到那个程度对扩充信源没有增量。

本次做法：

1. 先剔除已在 `x_capture_accounts` 和 `non_mainstream_media_sources` 中的信源
2. 再剔除竞品/Odaily 自己的站内链接
3. 重点看「腰部账号」（被引 2~50 次）和「编辑用过但机器没抓」的账号
4. 结论用「竞品引用数据 + X 时间线抽样」双数据源交叉验证

### 发现三：单一 AI 判断与编辑实际使用存在偏差

| 账号 | AI 判断 | 引用数据 | 最终结论 |
|---|---|---|---|
| BinanceWallet | 80% 营销，不值得 | 两个月被引 15 次，全是 Alpha 上新 | **值得**（Alpha 上新是用户刚需，建议走正则/固定模板） |
| ZachXBT | 样本不足 | 两个月被引 10 次，全是独家调查 | **值得**（低频高爆点型） |
| unusual_whales | 值得跟踪 | 仅被引 2 次，内容偏美股政治 | **观察，暂不接入** |

## 三、统计口径

| 项目 | 说明 |
|---|---|
| 数据来源 | Supabase `newsflash_items`（Odaily、律动、PANews、金色四家快讯） |
| 时间范围 | `published_at` 在过去 60 天内 |
| 「被引次数」 | 该信源作为**原文出处**（`source_url`）出现在快讯中的次数，**不是**账号自身发帖数 |
| X 账号拆解 | `x.com/账号名` → `x:账号名` |
| TG 频道拆解 | `t.me/频道名` → `tg:频道名` |
| 网站拆解 | 取域名，如 `site:bloomberg.com` |
| 已剔除 | 竞品站内页（jinse2.com、panewslab.com、theblockbeats.info 等）、Odaily 站内页、已在追踪列表的 X 账号和已接入外媒站点 |
| 金色缺口 | 金色仅在外部原文链接存在时保留 `source_url`，站内页为空，金色部分统计天然不全 |
| 脏数据 | 约 60 条链接格式异常（双重协议头等），已剔除 |
| 时间线抽样 | 通过 X 插件抓取各账号近期推文，由 AI 辅助分析发帖风格，与引用数据交叉验证 |
