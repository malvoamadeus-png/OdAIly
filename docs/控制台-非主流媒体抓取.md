# 控制台-非主流媒体抓取

## 说明

文件名仍保留旧实现名，但当前页面的业务标题是“外媒抓取控制台”。

它管理的是统一外媒收集器，对应：

- `non_mainstream_media_settings`
- `non_mainstream_media_sources`

## 页面能力

- 配置全局抓取频率
- 配置全局抖动
- 按站点启停
- 按 `pipeline_mode` 分组展示

## 分组口径

- `write_flow`：进入编写链路
- `alert_only`：标题提醒链路

当前这些已接入站点会这样出现：

- CoinDesk -> `write_flow`
- Cointelegraph -> `write_flow`
- Decrypt -> `write_flow`
- Tether News -> `write_flow`
- The Block -> `alert_only`

其中：

- Cointelegraph 的实际发现源优先走 `https://cointelegraph.com/sitemap/google-news.xml`
- Cointelegraph 的 `RSS` 只作为 fallback，不再是主发现入口
- Tether News 默认优先请求官方 `wp-json`；如果服务器侧被 Cloudflare challenge 拦住，会自动切到 `r.jina.ai` 代理后的官方页面，不需要人工改源
- The Block 默认优先请求官方 `rss.xml`；如果服务器侧直连被 Cloudflare 拦住，会自动切到 `r.jina.ai` 代理后的官方 RSS，只有两者都失败时才退回 Google News
- The Block 继续只走标题提醒，不进入正文编写链路

## 不做的事

- 不重命名数据库表
- 不展示 `mainstream_media` 历史队列
- 不在前端暴露已退役的 `external_media_alert_fetcher`
