# 控制台-非主流媒体抓取

## 职责

控制台-非主流媒体抓取用于维护收集者-非主流媒体的最小配置。

当前前端是现有 Vite 单页中的一个独立视图，标题为“非主流媒体抓取控制台”。

这页只做两件事：

- 配置全局抓取节奏
- 展示当前已接入站点并控制启停

## 读写范围

前端直接读写两张表：

- `non_mainstream_media_settings`
- `non_mainstream_media_sources`

其中：

- `non_mainstream_media_settings` 维护全局参数
- `non_mainstream_media_sources` 维护站点启停和最近状态

`non_mainstream_media_seen_items` 不暴露给控制台。

## 配置项

全局配置：

- `global_interval_seconds`：全局抓取频率，默认 `60`
- `jitter_seconds`：全局抖动秒数，默认 `5`

站点行展示字段：

- `site_key`
- `display_name`
- `homepage_url`
- `capture_method`
- `enabled`
- `seeded_at`
- `last_polled_at`
- `last_success_at`
- `last_error`

当前前端只允许修改：

- 全局频率
- 全局抖动
- 单站点 `enabled`

当前前端不允许修改：

- 站点新增/删除
- `site_key`
- `display_name`
- `homepage_url`
- `capture_method`

也就是说，`capture_method` 在控制台是只读展示，不是可编辑配置。

## 站点来源

站点目录由后端注册表定义，前端只展示当前后端已经接入的站点。

同步规则：

- `non-mainstream-media-init-db` 会把后端注册表同步到 `non_mainstream_media_sources`
- non-mainstream worker 启动时也会同步一次
- 已存在的站点行保留 `enabled`，只更新只读元数据

第一版默认预置：

- `a16z_crypto_posts`

## 页面边界

当前页面展示：

- 当前全局抓取频率
- 当前全局抖动
- 已接入站点列表
- 站点抓取方式
- 站点入口 URL
- 是否已 seed
- 最近轮询时间
- 最近成功时间或最近错误

当前页面不展示：

- 抓取日志
- 最近入库任务
- 详情页调试结果
- 站点抓取规则编辑
- Prompt 内容

这些内容分别属于日志、任务流水线或 Prompt 控制台，不放进这个页面。

## 生效机制

控制台更新 `non_mainstream_media_settings` 或 `non_mainstream_media_sources` 后，数据库会发出：

- `non_mainstream_media_config_changed`

收集者-非主流媒体 worker 会：

- 启动时主动读取当前配置
- 运行中监听 `LISTEN/NOTIFY`
- 收到通知后重新加载配置

因此这页的配置变更不要求重启 worker。
