# 控制台-非主流媒体抓取

## 说明

文件名仍保留旧实现名，但当前控制台拆成“外媒”和“AI信源”两个侧边栏标签。

它管理的是复用 `non_mainstream_media` 实现包的外媒 / AI信源收集器，对应：

- `non_mainstream_media_settings`
- `non_mainstream_media_sources`

worker 启动时会加载当前配置；运行中不再保留专门的 Postgres 监听连接，默认每 5 分钟内至少重载一次配置，不需要重启服务。

## 页面能力

- 配置全局抓取频率
- 配置全局抖动
- 按站点启停
- 按 `pipeline_mode` 分组展示
- “外媒”标签只展示 `source_group = 'external_media'` 的站点
- “AI信源”标签只展示 `source_group = 'ai_source'` 的站点
- 站点行展示有效抓取频率，优先使用站点级 `interval_seconds`
- 站点行只读展示发现策略 `discovery_mode`

## 分组口径

- `write_flow`：进入编写链路
- `alert_only`：标题提醒链路

当前这些已接入站点会这样出现：

- CoinDesk -> `write_flow`
- Cointelegraph -> `write_flow`
- Decrypt -> `write_flow`
- Tether News -> `write_flow`
- The Block -> `alert_only`
- TheElec CHINA -> AI信源标签下的 `write_flow`，站点级频率 `300s`
- ETNews Electronics -> AI信源标签下的 `write_flow`，站点级频率 `300s`
- ETNews SW -> AI信源标签下的 `write_flow`，站点级频率 `300s`
- ZDNet Korea Semiconductor -> AI信源标签下的 `write_flow`，站点级频率 `300s`
- CTEE Semiconductor -> AI信源标签下的 `write_flow`，站点级频率 `300s`
- Hankyung Premium9 -> AI信源标签下的 `write_flow`，站点级频率 `300s`

其中：

- CoinDesk / Cointelegraph / Decrypt 会展示 `Telegram 优先 · 直抓兜底`
- The Block 与其它现有站点会展示 `站点直抓`
- Cointelegraph 的实际发现源优先走 `https://cointelegraph.com/sitemap/google-news.xml`
- Cointelegraph 的 `RSS` 只作为 fallback，不再是主发现入口
- CoinDesk / Cointelegraph / Decrypt 除原站点直抓外，还由独立 `telegram-discovery-worker` 先轮询 `PhoenixNewsEN` 做提前发现；前端本次只展示，不提供在线切换
- Tether News 默认优先请求官方 `wp-json`；如果服务器侧被 Cloudflare challenge 拦住，会自动切到 `r.jina.ai` 代理后的官方页面，不需要人工改源
- TheElec CHINA 默认优先请求 `www.thelec.net` 的列表页和正文页；如果主域名不可达，会自动退到 `thelec.net`，必要时再退到 `r.jina.ai` 代理后的 `thelec.net`
- TheElec CHINA 的正文页在 `www` / 裸域都被拦时，会继续解析 `r.jina.ai` 返回的 Markdown 正文，不再只支持 HTML 正文
- TheElec CHINA 属于 AI信源，不会出现在外媒标签里
- ETNews Electronics / ETNews SW 属于 AI信源，不会出现在外媒标签里；列表页分别对应 `id1=06` 和 `id1=04`
- ZDNet Korea Semiconductor 属于 AI信源，不会出现在外媒标签里；列表页对应 `newskey/?lstcode=반도체`
- CTEE Semiconductor 属于 AI信源，不会出现在外媒标签里；列表页对应 `industry/semi`，当前必须使用 `www.ctee.com.tw`
- Hankyung Premium9 属于 AI信源，不会出现在外媒标签里；列表页对应 `premium9/0100001`
- Hankyung Premium9 在服务器侧如果被 Cloudflare 拦截，会自动切到 `r.jina.ai` 代理后的列表页 / 正文 Markdown
- The Block 默认优先请求官方 `rss.xml`；如果服务器侧直连被 Cloudflare 拦住，会自动切到 `r.jina.ai` 代理后的官方 RSS，只有两者都失败时才退回 Google News
- The Block 继续只走标题提醒，不进入正文编写链路

## 不做的事

- 不重命名数据库表
- 不展示 `mainstream_media` 历史队列
- 不在前端暴露已退役的 `external_media_alert_fetcher`
- 不在前端提供 `discovery_mode` 编辑能力
