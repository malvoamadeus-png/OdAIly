# 判断者-领域分类

## 职责

判断者-领域分类只服务外媒标题提醒链路，用来判断一条标题提醒是不是 Crypto 相关。

它不复用主写作链路里的 `regular / onchain / funding / non_mainstream_media` 判断契约，也不负责写作价值判断。

当前固定链路：

```text
收集者-非主流媒体(alert_only) -> 判断者-领域分类 -> 搜索者 -> 提醒者
```

## 输入

输入来自 `tasks.source = 'external_media_alert'` 的标题级任务。

这类任务稳定可拿到的信息只有：

- `site_display_name`
- `site_key`
- `title`
- `excerpt`
- `source_url`

## 输出

领域分类只允许两种结果：

```json
{
  "route": "crypto | discard"
}
```

落库后的业务含义：

- `route=crypto`：任务进入标题提醒链路的查重阶段。
- `route=discard`：任务终止为 `discarded`。

当前唯一的丢弃原因固定为：

- `discard_reason = non_crypto`

## 判断策略

当前采用两层判断：

1. 规则预筛
2. 模型兜底

### 规则预筛

直接保留：

- `ft_crypto`
- `fortune_crypto`

因为这两个入口本身就是加密栏目。

另外，如果标题、摘要或链接里明确命中强 Crypto 词，也直接保留，例如：

- `crypto`
- `bitcoin`
- `ethereum`
- `stablecoin`
- `token`
- `blockchain`
- `btc / eth`
- `比特币 / 以太坊 / 稳定币 / 区块链 / 代币`

如果标题或摘要明显是强非 Crypto 主题，且没有 Crypto 语境，则直接丢弃，例如：

- 体育
- 娱乐
- 旅游
- 美食
- 天气
- 泛 AI 科技但没有加密语境

### 模型兜底

规则无法明确判断时，才调用模型。

模型只看：

- 来源媒体
- 站点标识
- 标题
- 摘要
- 原链接

模型不输出解释文本，只输出结构化 JSON。

## Prompt 与版本

领域分类使用独立 Prompt：

- `template_key = external_media_alert_domain_judge`

当前初始 seed 文件：

- `docs/外媒标题领域判断模板.txt`

Prompt 版本由现有 Prompt 控制台统一管理，不放进“外媒”页。

## 运行配置

当前默认环境变量：

- `EXTERNAL_MEDIA_ALERT_DOMAIN_JUDGE_MODEL`
- `EXTERNAL_MEDIA_ALERT_OPENAI_BASE_URL`
- `EXTERNAL_MEDIA_ALERT_OPENAI_API_STYLE`

未单独配置时，会回退到现有 X 处理链路的 OpenAI 配置。

## 运行命令

```powershell
python backend\src\main.py external-media-alert-init-db
python backend\src\main.py external-media-alert-worker --stage domain_judge
python backend\src\main.py external-media-alert-worker --stage domain_judge --once
```

## 后续状态

- `pending -> classifying -> classified`
- `discarded`
- `domain_failed`

具体字段以 `external_media_alert_pipeline` 和当前 worker 实现为准。
