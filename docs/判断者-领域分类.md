# 判断者-领域分类

## 职责

判断者-领域分类服务两条外媒链路，用来判断一条外媒内容是不是 Crypto 相关，以及是否应被视为行情分析类噪音。

它不复用主写作链路里的 `regular / onchain / funding / non_mainstream_media` 判断契约，也不负责正文写作。

当前固定链路：

```text
收集者-非主流媒体(alert_only) -> 判断者-领域分类 -> 搜索者 -> 提醒者
收集者-外媒快讯 -> media_newsflash -> tasks(mainstream_media) -> 判断者-领域分类 -> 搜索者 -> 编写者1/编写者2
```

## 输入

输入来自以下两类任务：

- `tasks.source = 'external_media_alert'` 的标题提醒任务
- `tasks.source = 'mainstream_media'` 的主流外媒快讯任务

两类任务稳定可拿到的信息包括：

- `site_display_name`
- `site_key`
- `title`
- `excerpt`
- `source_url`

其中主流外媒快讯通常还会带上：

- `content`
- `original_title`
- `origin_source`

## 输出

领域分类只允许两种 route：

```json
{
  "route": "crypto | discard",
  "discard_reason": "none | non_crypto | market_analysis"
}
```

落库后的业务含义：

- `route=crypto`：任务进入后续查重阶段。
- `route=discard`：任务终止为 `discarded`。

当前丢弃原因有两类：

- `discard_reason = non_crypto`
- `discard_reason = market_analysis`

其中：

- `non_crypto`：内容不属于加密行业快讯。
- `market_analysis`：内容虽然与加密市场相关，但本质上是纯价格播报、技术分析、支撑阻力位描述或指数涨跌罗列，不进入 Odaily 快讯生成链路。

## 判断策略

当前所有外媒任务都直接调用 AI 判断，不再使用关键词、短 ticker 或站点名做确定性直通/丢弃。

这样做是为了避免 `SOL` / `ETH` / `BTC` 这类短词在普通英文单词、URL 路径或栏目名里误命中。例如 `solar`、`investing` 不能因为包含短片段就被直接判为 Crypto。

模型输入只包含：

- 来源媒体
- 站点标识
- 标题
- 摘要
- 原始快讯正文（如果有）
- 原链接

模型不输出解释文本，只输出结构化 JSON。

当前默认模型为 `gpt-5.4-mini`。

Prompt 需要额外明确屏蔽以下内容并输出 `discard_reason = market_analysis`：

- 单纯播报某币种、ETF 或板块涨跌
- 只给出支撑位、阻力位、目标价的技术分析
- 大段罗列指数、板块或市场表现，没有新增新闻事实

## Prompt 与版本

领域分类使用独立 Prompt：

- `template_key = external_media_alert_domain_judge`

当前初始 seed 文件：

- `docs/外媒标题领域判断模板.txt`

Prompt 版本由现有 Prompt 控制台统一管理，不放进“外媒”页。

## 运行配置

当前默认环境变量：

- `EXTERNAL_MEDIA_ALERT_DOMAIN_JUDGE_MODEL`
- `EXTERNAL_MEDIA_ALERT_OPENAI_BASE_URL`
- `EXTERNAL_MEDIA_ALERT_OPENAI_API_STYLE`

未单独配置时，会回退到现有 X 处理链路的 OpenAI 配置。

## 运行命令

```powershell
python backend\src\main.py external-media-alert-init-db
python backend\src\main.py external-media-alert-worker --stage domain_judge
python backend\src\main.py external-media-alert-worker --stage domain_judge --once
```

生产环境建议使用 `deploy/odaily-external-media-alert@.service`，实例名：

- `odaily-external-media-alert@domain_judge.service`

## 后续状态

- `pending -> classifying -> classified`
- `discarded`
- `domain_failed`

具体字段以 `external_media_alert_pipeline` 和当前 worker 实现为准。
