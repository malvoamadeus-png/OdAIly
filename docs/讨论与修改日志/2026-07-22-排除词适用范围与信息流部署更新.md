# 2026-07-22 排除词适用范围与信息流部署更新

## 背景

本次更新覆盖两组生产变更：

- 排除词规则组新增“适用范围”，从原先隐含的标题匹配升级为规则组级配置。
- 编辑插件信息流调整为高频区 / AI区两段展示，巨鲸信号并入高频区，此前消息不再单独占用侧边栏分区。

## 排除词适用范围

`source_exclusion_rule_groups` 新增 `match_target` 字段：

- `title`：仅标题类文本生效。
- `all`：标题、摘要和正文均生效。

默认值为 `title`，兼容已有排除词“只对标题生效”的产品口径。所有收集入口仍按既有 `x / competitor / crypto_source / ai_source / mixed_source / jin10` 路径选择规则组，URL、来源名、作者名和 metadata 不参与匹配。

各入口的文本边界：

- X：tweet 文本作为标题类文本。
- 竞品：标题为 `title`，正文为 `content`；Odaily reference 不走 `competitor` 排除。
- Crypto / AI / 混合信源：标题为发现页或文章标题，摘要和正文放入正文侧；混合信源仍分类前先查 `mixed_source`，分类后再查目标路径。
- 金十：标题为 `title`，正文为 `content`。

控制台排除词编辑页新增“适用范围”单选项，并在本地测试区拆分“测试标题”和“测试摘要 / 正文”，避免标题范围规则被正文测试文本误报命中。

## 信息流展示

编辑插件信息流从三段改为两段：

- 高频区：新快讯、Crypto信源标题提醒、审核者、巨鲸。
- AI区：AI信源消息。

巨鲸信号在本地 feed 写入和读取时统一归入高频区；历史兼容数据即使仍保存为 `low`，读取时也会规范化为 `high`。侧边栏仅展示 `high / ai`，并按时间线排序，不再为单一类型预留固定席位。

## 数据库与部署注意

`pipeline_schema.py` 的初始化 SQL 会：

- 新库建表时直接创建 `match_target text NOT NULL DEFAULT 'title'`。
- 旧库初始化时通过 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 补列。
- 增加 `match_target IN ('title', 'all')` 约束。
- 旧 `competitor_filter_keywords` 迁移出的兼容规则组默认写入 `match_target = title`。

生产部署顺序保持仓库规则：

1. 本地测试通过后提交并推送到 GitHub。
2. 生产 `/opt/OdAIly` 执行 `git fetch` 和 `git pull --ff-only`。
3. 执行相关 schema 初始化命令，让 `source_exclusion_rule_groups.match_target` 在生产库生效。
4. 重启读取排除词或信息流代码的常驻服务。
