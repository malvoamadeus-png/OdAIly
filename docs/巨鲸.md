# 巨鲸

## 职责

巨鲸模块是一条独立监控链路，不进入 `tasks` 写作发布流程。当前包含两块：

- `链上`：监控配置地址在启用 EVM 链上的转账和疑似 Swap。
- `Hyperliquid`：监控配置地址在 Hyperliquid 上的开仓、平仓成交。

两块都由控制台维护地址和标签，后端 worker 负责轮询、识别、去重、落库和 Telegram 播报。

## 前端输入

控制台新增一级标签“巨鲸”，页面内包含两个分区：

- `链上`：输入 `address + label`，支持启用、停用、删除，查看各链最近状态和播报历史。
- `Hyperliquid`：输入 `address + label`，支持启用、停用、删除，查看最近轮询状态和播报历史。

前端只维护地址级配置，不提供链选择，也不手工选择 Hyperliquid 合约或币种。

## 数据表

链上监控使用：

- `whale_watch_addresses`
- `whale_watch_chain_states`
- `whale_watch_activities`

Hyperliquid 监控使用：

- `whale_watch_hyperliquid_addresses`
- `whale_watch_hyperliquid_states`
- `whale_watch_hyperliquid_activities`

这些表都走 Supabase Auth + RLS。只有 `authenticated` 且存在于 `console_admins` 白名单中的用户可以读写地址配置，并读取状态与活动记录；`anon` 不授予权限。

## Worker 与命令

初始化数据库：

```powershell
python backend\src\main.py whale-watch-init-db
```

链上 worker：

```powershell
python backend\src\main.py whale-watch-worker
python backend\src\main.py whale-watch-worker --once
```

Hyperliquid worker：

```powershell
python backend\src\main.py whale-watch-hyperliquid-worker
python backend\src\main.py whale-watch-hyperliquid-worker --once
```

生产环境分别由 `deploy/odaily-whale-watch.service` 与 `deploy/odaily-whale-watch-hyperliquid.service` 托管，服务统一读取 `/opt/OdAIly/.env`。

## 链上监控来源与识别

链上监控按 `WHALE_WATCH_CHAIN_KEYS` 轮询启用链，第一版默认启用：

- `ethereum` -> `https://eth.blockscout.com`
- `base` -> `https://base.blockscout.com`

每轮按 `address + chain` 调用 Blockscout v2：

- `/api/v2/addresses/{address}/transactions`
- `/api/v2/addresses/{address}/token-transfers`
- `/api/v2/transactions/{tx_hash}`

首次看到某个 `address + chain` 组合且没有 `seeded_at` 时，只 seed 当前第一页视图，不播报历史交易。

活动识别规则：

- 只有单向净流入或净流出时，识别为普通转账。
- 同一笔交易内同时存在资产流出与另一资产流入，且命中常见 DEX / router / pool 信号时，识别为疑似 Swap。
- 识别结果按 `address_id + chain_key + tx_hash + activity_fingerprint` 去重。

第一版 Swap 识别是规则判定，不做完整 ABI 深度解码。

## Hyperliquid 监控来源与识别

Hyperliquid worker 直接调用官方 `https://api.hyperliquid.xyz/info`，当前使用：

- `userFills`

识别范围：

- `Open Long`
- `Open Short`
- `Close Long`
- `Close Short`

首次看到地址且没有 state 时，只记录当前 `last_seen_time`，不播报历史成交。

识别结果保存：

- 方向、币种、价格、数量、名义价值、已实现盈亏
- `fill_key`
- `fill_time` / `fill_time_ms`
- `telegram_text`
- `tx_url`

过滤规则：

- `notional_usd < WHALE_HYPERLIQUID_MIN_NOTIONAL_USD` 的成交不通知，默认阈值是 `50000` 美元。
- 低于阈值的成交不会写入活动表，只记入本轮 worker 统计。

Hyperliquid 活动按 `address_id + fill_key` 去重。

## Telegram

两条链路都复用现有 `TelegramClient`。

链上 topic 回落顺序：

```text
WHALE_TELEGRAM_MESSAGE_THREAD_ID
TELEGRAM_MESSAGE_THREAD_ID
```

Hyperliquid topic 回落顺序：

```text
WHALE_HYPERLIQUID_TELEGRAM_MESSAGE_THREAD_ID
WHALE_TELEGRAM_MESSAGE_THREAD_ID
TELEGRAM_MESSAGE_THREAD_ID
```

发送结果会写回活动表的 `telegram_result`。发送失败时，活动仍会落库避免重复刷屏，同时状态表会记录 `last_error` 供控制台排查。

## 配置

链上：

```text
WHALE_WATCH_INTERVAL_SECONDS=60
WHALE_WATCH_CHAIN_KEYS=ethereum,base
WHALE_WATCH_REQUEST_TIMEOUT_SECONDS=20
WHALE_WATCH_MAX_ATTEMPTS=3
WHALE_WATCH_BACKOFF_SECONDS=1
WHALE_TELEGRAM_MESSAGE_THREAD_ID=
```

Hyperliquid：

```text
WHALE_HYPERLIQUID_TELEGRAM_MESSAGE_THREAD_ID=
WHALE_HYPERLIQUID_INTERVAL_SECONDS=60
WHALE_HYPERLIQUID_MIN_NOTIONAL_USD=50000
WHALE_HYPERLIQUID_REQUEST_TIMEOUT_SECONDS=20
WHALE_HYPERLIQUID_MAX_ATTEMPTS=3
WHALE_HYPERLIQUID_BACKOFF_SECONDS=1
```

公共 Telegram 环境变量：

```text
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
TELEGRAM_MESSAGE_THREAD_ID=
```
