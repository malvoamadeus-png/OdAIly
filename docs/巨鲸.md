# 巨鲸

## 职责

巨鲸模块用于维护需要关注的 EVM 地址，并在地址出现新的链上活动时向 Telegram 播报。

当前版本只实现“链上”监控；`Hyperliquid` 仅在前端保留占位，不建表、不启动 worker。

## 前端输入

控制台新增一级标签“巨鲸”，页面内包含：

- `链上`：输入 EVM 地址和自定义标签，支持启用、停用、删除。
- `Hyperliquid`：显示待接入占位。

前端只保存地址级配置，不选择链。后端会按 `WHALE_WATCH_CHAIN_KEYS` 对所有启用链检查同一个地址。

## 数据表

- `whale_watch_addresses`：地址配置，包含 `address`、`address_lower`、`label`、`enabled`。
- `whale_watch_chain_states`：按 `address_id + chain_key` 记录 seed、最近轮询、最近成功、错误和最后区块。
- `whale_watch_activities`：记录已识别活动，按 `address_id + chain_key + tx_hash + activity_fingerprint` 去重。

控制台通过 Supabase Auth 直连这些表。RLS 只允许 `authenticated` 且存在于 `console_admins` 的用户操作地址配置、读取状态和活动记录。

## Worker

初始化：

```powershell
python backend\src\main.py whale-watch-init-db
```

运行：

```powershell
python backend\src\main.py whale-watch-worker
python backend\src\main.py whale-watch-worker --once
```

生产环境使用 `deploy/odaily-whale-watch.service` 托管长期 worker，服务读取 `/opt/OdAIly/.env`。

worker 每轮读取启用地址和启用链，对每组 `address + chain` 调用 Blockscout v2：

- `/api/v2/addresses/{address}/transactions`
- `/api/v2/addresses/{address}/token-transfers`
- `/api/v2/transactions/{tx_hash}`

首次看到某地址在某条链上没有 `seeded_at` 时，只记录当前页最高区块为已见，不播报历史活动。

## 活动识别

普通转账：

- 只有原生币或 token 净流入时，播报“收到”。
- 只有原生币或 token 净流出时，播报“转出”。

Swap：

- 同一笔交易里该地址同时存在资产流出和另一资产流入。
- 交易方法、目标合约名、转账对手标签命中常见 DEX/router/pool 信号，例如 `execute`、`swap`、`UniversalRouter`、`Uniswap`、`PoolManager`。
- 播报格式类似：`「自定义标签」在 Base 用 0.1 ETH 换入 43.8542 PITCH`。

第一版是规则识别，不做全协议 ABI 深度解码；目标是覆盖常见 DEX/router 交易并通过活动唯一键避免重复播报。

## Telegram

复用现有 `TelegramClient`。

环境变量：

```text
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
WHALE_TELEGRAM_MESSAGE_THREAD_ID=
TELEGRAM_MESSAGE_THREAD_ID=
```

`WHALE_TELEGRAM_MESSAGE_THREAD_ID` 优先；未配置时回落到 `TELEGRAM_MESSAGE_THREAD_ID`。

发送结果写入 `whale_watch_activities.telegram_result`。发送失败时，活动仍会入库并避免重复刷屏，同时对应链状态会留下 `last_error` 便于控制台排查。

## 配置

```text
WHALE_WATCH_INTERVAL_SECONDS=60
WHALE_WATCH_CHAIN_KEYS=ethereum,base
WHALE_WATCH_REQUEST_TIMEOUT_SECONDS=20
WHALE_WATCH_MAX_ATTEMPTS=3
WHALE_WATCH_BACKOFF_SECONDS=1
```

默认支持链：

- `ethereum` -> `https://eth.blockscout.com`
- `base` -> `https://base.blockscout.com`

新增 EVM 链时，先确认目标链有兼容 Blockscout v2 的公开实例，再扩展后端链注册表。
