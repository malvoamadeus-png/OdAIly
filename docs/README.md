# OdAIly 文档索引

`docs/` 只保留索引和三类目录。代码、业务流程或 Prompt 规则变化时，以实际代码和“OdAIly代码与整体设计”中的对应模块文档为准。

## `OdAIly代码与整体设计/`

可随代码提交到 GitHub。这里保存当前生产流的系统契约、收集者与处理阶段边界、控制台与插件说明、Prompt 种子、模板及 UI 设计。

- 系统入口：`完整程序架构.md`、`本地优先与Supabase档案库架构.md`。
- 处理阶段：`判断者.md`、`搜索者.md`、`编写者1.md`、`编写者2.md`、`编写者3.md`、`发布者.md`、`审核者.md`、`提醒者.md`、`监督者.md`。
- 收集与来源：全部 `收集者-*.md`、`巨鲸.md`、`MSX快讯.md`。
- 控制台与插件：`控制台*.md`、`信息流插件.md`、`快讯生成插件.md`、`UI风格.md`。
- 模板：全部 `*.txt` Prompt 种子和快讯模板。

## `讨论与修改日志/`

可随代码提交到 GitHub。这里保存已经发生的测试、调研、方案比较、阶段计划、样本分析和最终讨论结论；它们不是当前生产契约。

- 根目录：按日期记录的讨论结论和改动背景。
- `调研与测试/`：API 替换效果测试、Supabase 优化调研、GitHub 工具与组件调研、初始架构设计。
- `发布者及上下游改进计划/`：发布者相关的阶段计划、样本汇总和初筛结果。

公开讨论文档不得写入真实密钥、token、账号、主机、私有服务地址或一次性访问方式。历史材料中的路径、参数或旧方案只用于理解当时决策，不代表当前部署配置。

## `工具与运维私有/`

本机私有目录，已由 `.gitignore` 忽略，不提交到 GitHub。用于 Linux / Codex 访问、部署操作、连接槽与 session 排障、临时运维命令和其他可能包含环境细节的资料。

- 现有资料：`CODEX_LINUX_ACCESS.md`、`agent-operations-runbook.md`、`Supabase连接槽与Session排障.md`。
- 新增此类资料一律放在这里；如需公开通用原则，应提炼为不包含部署细节的新文档再放入前两类目录。
