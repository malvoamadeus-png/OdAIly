# Push Data 接口文档

## 1. 接口说明

该接口用于推送一条数据，包含标题、内容以及是否发布状态。

- **请求方式**：`POST`
- **请求地址**：`http://47.113.217.70:8501/push/data`
- **Content-Type**：`application/json`

---

## 2. 请求参数

### 请求体 JSON 结构

```json
{
  "title": "标题内容",
  "content": "正文内容",
  "isPublish": false
}
```

### 参数说明

| 参数名 | 类型 | 是否必填 | 默认值 | 说明 |
|---|---|---:|---|---|
| title | String | 是 | - | 推送标题 |
| content | String | 是 | - | 推送内容 |
| isPublish | Boolean | 否 | true | 是否发布，`true` 表示发布，`false` 表示不发布 |

---

## 3. Java 对象定义

```java
public class PushDataBO {

    private String title;
    private String content;
    private Boolean isPublish = Boolean.TRUE;
}
```

说明：

- `isPublish` 字段默认值为 `true`
- 如果调用方未传该字段，服务端可按默认发布处理
- 当前示例中实际传入的是 `false`

---

## 4. 请求示例

### Curl 示例

```bash
curl -X POST "http://47.113.217.70:8501/push/data" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "测试标题",
    "content": "测试内容",
    "isPublish": false
  }'
```

### Java 调用示例

```java
String urlPrefix = "http://47.113.217.70:8501/";
HttpRequest httpRequest = HttpUtil.createPost(urlPrefix + "push/data");

JSONObject data = new JSONObject();
data.put("title", title);
data.put("content", content);
data.put("isPublish", Boolean.FALSE);

httpRequest.body(data.toJSONString());
```

---

## 5. 请求示例说明

本次示例请求体等价于：

```json
{
  "title": title,
  "content": content,
  "isPublish": false
}
```

其中：

- `title`：业务传入的标题
- `content`：业务传入的内容
- `isPublish`：固定传 `false`

## 6. 返回结果

你提供的代码中**未包含响应结果定义**，因此接口返回结构暂无法准确确认。  
如果后端有统一返回格式，通常可能类似：

```json
{
  "code": 200,
  "msg": "success",
  "data": null
}
```

### 返回参数说明（示例）

| 参数名 | 类型 | 说明 |
|---|---|---|
| code | Integer | 状态码 |
| msg | String | 返回消息 |
| data | Object | 返回数据 |

> 注意：以上返回结构为**示例格式**，请以实际接口返回为准。

---

## 7. 注意事项

- 请求体格式必须为 `JSON`
- `title` 和 `content` 建议做非空校验
- `isPublish` 不传时，默认值为 `true`
- 如果接口服务部署在公网，建议补充鉴权机制，如：
  - Token
  - Sign 签名
  - AppKey / AppSecret
