# 收集者-外媒快讯

## 职责

收集者-外媒快讯负责持续拉取四家主流外媒的最新快讯/新闻标题，写入 Supabase 的 `media_newsflash`，并派生主流外媒处理任务。

当前它负责：

- 来源采集
- 基础清洗
- 站内去重
- 写入 `media_newsflash`
- 写入 `tasks.source = 'mainstream_media'`

当前它不负责：

- 直接调用 `external_media_alert` 的 AI 判断或查重
- 直接发送 Telegram
- 与 Odaily 或历史草稿做跨来源去重
- 直接生成后台待审核内容

## 当前实现范围

当前固定接入四个来源：

- `coindesk`
  - 主入口：`https://www.coindesk.com/arc/outboundfeeds/rss/`
  - 回退入口：`https://www.coindesk.com/`
- `cointelegraph`
  - 主入口：`https://cointelegraph.com/rss`
  - 回退入口：`https://cointelegraph.com/category/latest-news`
- `the_block`
  - 主入口：Google News RSS `site:theblock.co crypto`
  - 回退入口：`https://www.theblock.co/latest-crypto-news`
- `decrypt`
  - 主入口：`https://decrypt.co/feed`
  - 回退入口：`https://decrypt.co/news`

优先策略固定为：

- 先抓 RSS
- RSS 失败或为空时，再抓列表页 HTML

## 存储边界

### Supabase / Postgres

当前权威落库表为：

- `media_newsflash`
- `tasks`

当前收集者写入字段：

- `source`
- `title`
- `content`

同时内部保留：

- `source_url`
- `title_key`
- `published_at`
- `metadata`
- `created_at`
- `updated_at`

其中 `source_url` 和 `title_key` 只用于采集侧去重与排障，不改变前端可继续读取的主字段语义。

同一批落库中，只有成功写入 `media_newsflash` 的新条目才会派生一条 `tasks` 记录；命中重复规则的条目直接忽略，不再创建新的 `mainstream_media` 任务。派生任务的关键契约如下：

- `tasks.source = 'mainstream_media'`
- `tasks.source_item_id` 优先使用 `source + 归一化 source_url`，没有可用 URL 时回退为 `source + title_key`
- `tasks.title = media_newsflash.title`
- `tasks.content = media_newsflash.content`
- `tasks.metadata` 至少包含 `source_kind`、`origin_source`、`media_source_item_id`、`original_title`、`excerpt`、`site_display_name`

如果上游来源拿不到可靠发布时间，当前会用采集落库时刻回填 `published_at`，避免后续统一时效门因为缺时间直接丢弃任务。

### 本地运行态

当前没有单独的本地 SQLite 工作集。

该收集者直接请求源站并写入 Supabase。

## 去重契约

同一条快讯不重复入库。

当前去重口径：

- 同一 `source` 下，`source_url` 相同则视为重复
- 如果 URL 不可用或发生变化，`title` 归一化后的 `title_key` 相同也视为重复

命中上述任一重复规则时：

- 不新增 `media_newsflash`
- 不派生新的 `tasks.source = 'mainstream_media'`

归一化规则只用于去重，不回写覆盖原始标题。

## 轮询与失败处理

当前固定轮询间隔：

- `40` 秒

单站点处理结果分为：

- `success`
- `fetch_failed`
- `parse_empty`

失败不会中断其他站点；当轮失败会在下一轮继续重试。

## 心跳

收集者会写 `pipeline_worker_heartbeats`，组件名固定为：

- `external_media_alert_fetcher`

与其他主要 worker 一样：

- 稳定态最多 60 秒一条
- 状态变化立即写
- 失败立即写

## 运行命令

```powershell
python backend\src\main.py external-media-alert-init-db
python backend\src\main.py external-media-alert-fetcher --once
python backend\src\main.py external-media-alert-fetcher
```

注意：

- 这只初始化并运行主流外媒快讯的收集入口。
- `mainstream_media` 后续仍依赖 `external-media-alert-worker --stage domain_judge`、`external-media-alert-worker --stage search`、`x-process-worker --stage write`、`x-process-worker --stage format_publish`。
- 外媒标题提醒链路的 Telegram 发送仍由 `external-media-alert-worker --stage notify` 独立负责。
- 生产环境建议使用 `deploy/odaily-external-media-alert-fetcher.service` 常驻运行收集者。
