# 收集者-外媒快讯

## 当前状态

这份文档保留给历史兼容链路。

原先独立的主流外媒快讯入口 `external_media_alert_fetcher` 已退役，不再负责新的生产入库。CoinDesk、Cointelegraph、Decrypt、Tether News、The Block 已并入统一 Crypto信源收集器：

- CoinDesk -> `write_flow`，发现入口为 `Telegram 优先 + 站点直抓兜底`
- Cointelegraph -> `write_flow`，发现入口为 `Telegram 优先 + 站点直抓兜底`
- Decrypt -> `write_flow`，发现入口为 `Telegram 优先 + 站点直抓兜底`
- Tether News -> `write_flow`
- The Block -> `alert_only`，发现入口为 `Telegram 优先 + 站点直抓兜底`

## 历史兼容边界

- `mainstream_media` 任务仍可能存在于历史数据中
- `local_pipeline` 的 `alert_only` 链路仍兼容复用 `external_media_alert` 的 `domain_judge/search/notify` 阶段逻辑
- 但新任务不再通过 `media_newsflash -> mainstream_media` 这条路径产生

## The Block 入口

- 现役统一收集器现在优先用 Telegram 频道 `https://t.me/PhoenixNewsEN` 发现 The Block 新标题
- 原官方 `rss.xml` 仍保留为直抓兜底：`https://www.theblock.co/rss.xml`
- 如果服务器侧直连官方 RSS 被 Cloudflare 拦截，会自动 fallback 到 `r.jina.ai` 代理后的官方 RSS
- 只有当官方 RSS 和代理官方 RSS 都不可用时，才会退回 `Google News RSS`：`https://news.google.com/rss/search?q=site:theblock.co&hl=en-US&gl=US&ceid=US:en`
- Google News fallback 命中后会先解码回官方 `theblock.co/post/...` 链接，再按官方链接查重；过旧条目会按新鲜度窗口丢弃，避免旧标题反复提醒
- `external_media_alert_fetcher` 这条历史入口虽然保留代码兼容，但不再承担生产职责
- The Block 正文抓取仍不可用，因此 Telegram 命中后也只保留标题提醒用途
- 当前生产里，这类 `alert_only` 结果除了 Telegram 通知，也会进入插件信息流的高频区

## The Block Telegram 测试脚本

- 仓库新增独立测试脚本 `tools/telegram_channel_monitor_test.py`，用于验证 Telegram 频道 `https://t.me/PhoenixNewsEN` 是否能比现有 The Block RSS 更快发现新内容。
- 该脚本只做实验，不接入 `non_mainstream_media`、`external_media_alert`、本地流水线或任何生产 worker。
- 运行方式沿用 `D:\Coding\CommunityMonitor` 的 Telethon 方法：
  - 默认读取同目录 sibling 仓库 `CommunityMonitor/配置.txt` 里的 `App api_id/api_hash`
  - 默认复用 `CommunityMonitor/telegram_probe.session`
  - 默认每 `30` 秒轮询一次，首轮只 seed 当前最新消息，不把历史消息当成命中
  - 默认运行 `120` 分钟，并把 `heartbeat / new_message / flood_wait / finished` 事件写入 `data/raw/telegram_channel_test/*.log`
- 若检测到新消息，脚本会：
  - 在终端输出 `new_message`
  - 更新本地 state 文件，避免重复提示
  - 默认触发一次终端 bell；需要静音时可加 `--no-bell`
- 默认依赖不并入生产 `backend/requirements.txt`；测试依赖单独放在 `tools/telegram_channel_monitor_test_requirements.txt`
- 推荐命令：
  - Windows PowerShell：`py -m pip install -r tools\telegram_channel_monitor_test_requirements.txt`
  - Windows PowerShell：`py tools\telegram_channel_monitor_test.py --reset-state`

## Telegram-first 正文探针

- 另新增独立测试脚本 `tools/telegram_writeflow_probe.py`，用于验证 `CoinDesk / Cointelegraph / Decrypt` 这类 `write_flow` 站点在“频道先提示、正式发现层后到”的情况下，是否已经能直接抓正文。
- 该脚本只做实验，不入库、不排队、不触发本地流水线。
- 它会从频道消息中识别目标来源前缀、抽取详情链接，并直接调用现有 `non_mainstream_media.fetch_article(...)`。
- 这条探针已验证：Telegram-first 对部分 `write_flow` 站点可以直接拿到正文，不需要等待第三方缓存路径。
- 推荐先跑历史回扫：
  - Windows PowerShell：`py tools\telegram_writeflow_probe.py --mode backscan --backscan-limit 200`
